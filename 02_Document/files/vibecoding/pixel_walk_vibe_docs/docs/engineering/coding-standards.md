# 编码规范 Coding Standards

## TypeScript

- 开启 strict；
- 不使用无理由 `any`；
- 外部输入必须校验；
- 内容模型集中定义；
- 公共函数写返回类型；
- 场景配置使用只读常量。

## 组件

- 职责单一；
- 内容组件与场景组件分离；
- 避免超大组件；
- 副作用集中；
- 所有监听器清理；
- 动态模块有错误边界；
- 不把静态内容硬编码进首页场景。

## 动画

- 一个统一主滚动进度；
- 映射函数写纯函数；
- 环境循环与滚动状态分开；
- 粒子复用和固定随机种子；
- 不每帧创建大量对象；
- 不在多个 scroll handler 中读写布局。

## CSS

- 使用设计 Token；
- 避免魔法颜色；
- 像素尺寸尽量整数；
- hover 不改变布局；
- focus-visible 明确；
- 不用大量绝对定位承载正文。

## 文件命名

```text
PascalCase.tsx      组件
camelCase.ts        工具
kebab-case.md       文档
*.config.ts         配置
*.test.ts           单元测试
*.spec.ts           E2E或行为测试
```

## 错误处理

不吞异常；用户可见错误有友好提示；生产环境不暴露秘密；场景失败降级静态背景；演示失败不影响详情内容。

## Git

小提交、一个提交一个目标、不提交秘密和缓存、不修改无关文件、不用破坏性命令覆盖用户改动。
