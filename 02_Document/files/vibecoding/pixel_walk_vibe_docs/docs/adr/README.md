# Architecture Decision Records

ADR 用于记录重要且难以从代码中理解的决策。

命名：`NNNN-short-title.md`

状态：`Proposed`、`Accepted`、`Rejected`、`Superseded`、`Deprecated`。

模板：

```markdown
# ADR NNNN: 标题

- Status:
- Date:
- Decision Owners:

## Context

## Decision

## Alternatives

## Consequences

## Verification
```

新 ADR 不应重写旧 ADR 历史。若决策变化，新增 ADR 并标注 supersedes。
