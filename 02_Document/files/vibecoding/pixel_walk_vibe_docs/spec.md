# 产品规格说明书 Spec

版本：1.0.0  
状态：当前需求基线  
产品名称：Pixel//Walk（可在站点配置中替换）

## 1. 产品愿景

建设一个具有强烈个人辨识度的纯前端个人网站。访问者可以阅读博文、浏览并体验站点所有者本人编写的程序，并通过陆地、深海、星空三个连续世界了解站点所有者的作品与兴趣。

## 2. 用户

### 访客

招聘者、同行开发者、朋友、文章读者和程序体验者。

### 站点维护者

需要通过 Markdown/MDX 添加文章和程序、更新状态与截图、修改介绍，并使用静态托管上线。

## 3. 信息架构

推荐公开路由：

```text
/                     沉浸式首页
/articles             文章列表
/articles/[slug]      文章详情
/programs             “做点啥呢”程序列表
/programs/[slug]      程序详情与演示
/about                关于我
/404                  404
```

兼容要求：若现有版本使用 `/projects`，迁移后保留 `/projects` → `/programs` 的静态重定向或兼容页。导航显示名称始终为“做点啥呢”。

## 4. 首页叙事

### 阶段 A：明亮陆地

内容：个人名称、一句话介绍、文章入口和精选文章。

视觉：天空、云、山丘、草地、原创像素引导物和前中后景视差。

### 过渡 A：陆地下潜

```text
天空变暗
→ 底部出现水面
→ 水位上涨
→ 卡片经过水线产生短暂折射
→ 水覆盖屏幕
→ 镜头向下
→ 进入海底
```

### 阶段 B：深海程序档案

内容：“做点啥呢”、精选程序、程序状态、技术栈和体验入口。

视觉：深海渐变、光束、气泡、鱼群、海草、珊瑚、水母和海底档案终端。

### 过渡 B：海洋平滑过渡为星空

```text
水流增强
→ 镜头向水面上升
→ 浪花占据屏幕
→ 深海色、光束和透明度连续过渡
→ 泡沫成为星点
→ 气泡成为星尘
→ 水面高光成为银河
→ 星空稳定
```

Canvas 背景世界在全部进度和动效模式下保持水平，不得旋转、翻转或倾斜。不得出现船，也不得复刻电影画面。

### 阶段 C：星空

内容：关于我、技能、联系入口、重启旅程和文章/程序入口。

视觉：深色星空、星云、银河、星点、流星、星座和漂浮信号站。

## 5. 文章系统

### 文章列表

- 发布时间倒序；
- 精选文章；
- 标签筛选；
- 标题和摘要搜索；
- 阅读时间；
- 空状态；
- 响应式布局。

### 文章详情

- 标题、摘要、日期、标签；
- 目录；
- Markdown/MDX；
- 代码高亮、图片、表格和引用；
- 上一篇和下一篇；
- SEO 元数据。

正文不强制使用像素字体。

## 6. “做点啥呢”程序系统

### 定义

每个条目必须代表站点所有者本人编写的程序或演示。

程序类别：

```text
web-app
desktop-app
mini-program
tool
visualization
game-prototype
automation
experiment
other
```

程序状态：

```text
completed
in-progress
prototype
archived
```

演示类型：

```text
static-embedded
external-live
video
gif
screenshots
none
```

### 程序列表

显示程序名称、说明、封面、技术栈、类别、状态、演示类型、查看详情、立即体验和可选源码入口。

### 程序详情

包含：

- 程序解决的问题；
- 使用方式；
- 核心功能；
- 本人完成内容；
- 技术方案；
- 关键难点；
- 截图与演示；
- 当前限制；
- 后续计划；
- 源码地址；
- 隐私与数据说明。

### 演示诚实性

若只是前端演示，必须标注：

```text
这是静态演示版。
部分后端、登录、支付、数据库或实时功能未接入。
```

不得模拟成功接口后让访客误以为功能已经完整上线。

## 7. 程序演示安全

站内演示优先采用独立静态路由、沙箱 iframe 或外部新标签页。

iframe 使用最小权限，例如：

```html
sandbox="allow-scripts allow-forms allow-modals"
```

只有确实需要时才添加 `allow-same-origin`、`allow-popups` 或 `allow-downloads`。

## 8. 视觉风格

- 二维高精细像素风；
- 16-bit/32-bit 主机时代质感；
- 可爱但不幼稚；
- 色彩丰富；
- 轮廓清晰；
- 允许渐变、光晕和透明粒子；
- 禁止体素、三维方块和“我的世界”风格。

像素字体用于标题、短标签、按钮和导航；正文使用高可读中文字体。

## 9. 响应式

桌面提供完整视差、完整海空颜色/光束/粒子过渡、鼠标互动和星座交互，世界方向固定为 0°。

移动端保留三个世界，减少粒子，简化折射，世界方向同样固定为 0°，程序卡片单列，不依赖 hover，不出现横向滚动。

## 10. Reduced Motion

开启后：

- 世界方向固定为 0°，不执行快速视差；
- 水位上涨改为像素遮罩或短淡入；
- 三个世界依然有不同的静态场景；
- 内容顺序和功能不变。

## 11. 静态部署

生产构建必须输出静态目录，生成所有文章和程序详情，不需要常驻 Node，支持 GitHub Pages 或 Cloudflare Pages，刷新详情页不报 404。

## 12. SEO

每页提供唯一 title、description、canonical、Open Graph、正确语义结构、sitemap 和 robots.txt。RSS 推荐实现。

## 13. 非目标

第一阶段不实现登录注册、自建评论后端、数据库后台、在线支付、多人实时协作、复杂服务端搜索或必须 SSR 的功能。

## 14. 成功标准

1. 三个世界不看文字也能分辨；
2. 陆地到深海连续；
3. 深海到星空自然；
4. 向上滚动可倒放；
5. 文章可通过 Markdown/MDX 添加；
6. 程序示例均为本人编写；
7. 程序演示有明确状态和限制；
8. 详情页可直接访问和刷新；
9. 静态构建成功；
10. 移动端和 Reduced Motion 可用；
11. Canvas 失败时内容可访问；
12. 没有密钥和侵权素材；
13. 构建、测试和验收有记录。
