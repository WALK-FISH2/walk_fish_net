# Pixel//Walk 个人网站：Vibe Coding 文档体系

本目录是一套面向 **Codex、AI 编程代理与人类开发者共同协作** 的项目文档。

项目目标：

- 展示个人博文；
- 展示“做点啥呢”板块；
- “做点啥呢”专门展示站点所有者本人编写的程序示例、交互演示和实验作品；
- 首页采用陆地 → 深海 → 浪花翻转 → 星空的可逆滚动叙事；
- 使用精细二维像素风，而不是体素或“我的世界”风格；
- 第一阶段可构建并部署为纯静态网站。

> 当前状态说明：现有代码据 Codex 汇报已完成第一轮骨架，包括三段滚动框架、文章/项目内容系统、详情页、搜索筛选、响应式、Reduced Motion、SEO 与降级模式。本套文档没有直接审查现有仓库，因此这些状态均记为“已汇报完成，待独立验证”。

## 文档阅读顺序

AI 代理开始工作前，必须按顺序阅读：

1. [`constitution.md`](./constitution.md)：不可违反的项目原则；
2. [`spec.md`](./spec.md)：产品需求、页面功能、视觉目标和验收；
3. [`architecture.md`](./architecture.md)：技术边界和模块划分；
4. [`plan.md`](./plan.md)：阶段计划；
5. [`tasks.md`](./tasks.md)：当前任务与状态；
6. [`AGENTS.md`](./AGENTS.md)：AI 代理操作手册；
7. [`docs/adr/`](./docs/adr/)：关键架构决策。

## 核心名词

### Articles

站点所有者发布的博文、技术笔记、开发记录和个人思考。

### Programs / 做点啥呢

这是网站对外展示的栏目名称，对应领域模型 `Program`。它表示：

- 站点所有者本人编写的程序；
- 可在网页中体验的静态交互示例；
- 外部部署的个人程序；
- 前端功能演示；
- 视频、GIF 或截图形式的程序演示；
- 尚未完整上线，但值得记录的个人实验。

它**不是**泛化的公司项目列表、工作履历列表或第三方项目聚合。

代码与目录中统一优先使用：

```text
program
programs
program-demo
```

若现有代码已经使用 `/projects`，迁移时应提供兼容跳转，避免链接失效。

## 文档职责

| 文件 | 作用 | 何时更新 |
|---|---|---|
| `constitution.md` | 不可轻易改变的项目原则 | 原则正式修订时 |
| `spec.md` | 产品与体验的事实来源 | 需求变化时 |
| `architecture.md` | 当前技术结构和边界 | 架构变化时 |
| `plan.md` | 阶段和里程碑 | 阶段变化时 |
| `tasks.md` | 可执行任务与状态 | 每次开发会话 |
| `AGENTS.md` | AI 代理操作手册 | 协作方式变化时 |
| `docs/adr/*.md` | 记录关键决策及理由 | 做出难逆决策时 |
| `CHANGELOG.md` | 已发布变化 | 每次版本发布 |

## 任务状态标记

```text
[ ] 未开始
[~] 已汇报完成，但尚未独立验证
[x] 已完成且有验证证据
[!] 阻塞
[-] 取消或不再需要
```

AI 代理不能仅凭“代码看起来正确”把任务标为 `[x]`。必须有构建、测试、浏览器验证或人工验收证据。

## 推荐给 Codex 的开场提示

```text
请先阅读 constitution.md、spec.md、architecture.md、plan.md、tasks.md 和 AGENTS.md。
不要立即修改代码。先检查仓库状态、package.json、当前任务和相关 ADR，
然后给出本轮最小实施计划。
```

开发完成后的最小汇报：

```text
1. 完成了什么；
2. 修改了哪些文件；
3. 执行了哪些命令；
4. 测试结果；
5. 是否更新了tasks.md和相关文档；
6. 仍存在什么风险；
7. 下一步建议。
```
