# 第二轮开发任务：将当前像素UI骨架升级为完整的像素叙事场景

请先完整阅读现有项目代码、README、原始需求文档以及当前页面实现。

- 首页三阶段滚动框架；
- 可逆滚动；
- 导航；
- Reduced Motion；
- SEO与基础降级模式。

本轮不要重新创建项目，不要推翻现有内容系统，不要重新设计路由。

本轮目标是：

把当前“纯色背景 + 矩形卡片”的第一轮骨架，升级为真正具有场景、层次、叙事和环境动画的二维高精细像素网站。

当前截图中的主要问题是：
1. 陆地、深海和星空几乎都使用同一种亮蓝色背景；
2. 场景主要依赖标题和卡片颜色区分；
3. 缺少云、山、草地、海水、鱼群、珊瑚、浪花、星云等环境；
4. 缺少连续的水位上涨、下潜、上升、海空平滑过渡和泡沫转星点过程；
5. 卡片像普通企业网站，没有融入游戏世界；
6. 页面留白很大，但没有被场景和视差元素利用。

请按照以下要求逐步实施。

## 一、总体视觉方向

目标风格：

- 二维横版像素游戏；
- 精细的16-bit、32-bit主机时代美术质感；
- 可爱但不幼稚；
- 色彩丰富；
- 场景有前景、中景、远景；
- 像素轮廓清晰；
- 可以有平滑渐变、透明光束和光晕；
- 不能做成体素、方块或“我的世界”风格；
- 不能只用CSS矩形和大面积纯色假装像素场景。

不要使用：
- 超级玛丽原版角色；
- 任天堂素材；
- 加勒比海盗素材；
- 第三方游戏地图；
- 来源不明的像素素材；
- 带版权风险的音效和字体。

本轮可以优先使用原创程序化像素图形、原创SVG、Canvas绘制和自行设计的精灵占位，但这些占位必须已经具备完整的场景构图，不能继续只使用矩形和圆形作为最终画面。
