# 项目宪章 Constitution

版本：1.0.0  
状态：生效  
适用范围：Pixel//Walk 个人网站全部源码、内容、素材和自动化流程。

本文件定义项目中不可被普通任务随意突破的原则。任何代码、设计、提示词或临时实现均不得与本宪章冲突。

## 第一条：静态优先

网站第一阶段必须能够构建为纯静态资源：

- 核心页面输出为 HTML、CSS、JavaScript 和静态资源；
- 文章和程序详情页在构建阶段生成；
- 部署后不需要 Node 服务端常驻运行；
- 不依赖数据库；
- 不依赖传统后端接口才能阅读核心内容；
- GitHub Pages、Cloudflare Pages、Netlify 或同类平台可托管。

除非通过 ADR 正式批准，否则不得引入强制 SSR、运行时数据库、必须常驻的 Node 服务或无法静态导出的核心路由。

## 第二条：“做点啥呢”必须代表本人编写的程序

允许展示：

- 用户本人开发的程序、前端演示、工具、算法可视化和交互实验；
- 用户本人参与开发且能够明确说明个人职责的程序。

不得将其模糊为：

- 泛化的公司项目宣传；
- 第三方项目集合；
- 未参与开发的产品；
- 虚构功能；
- 仅有概念图却宣称已经完成的程序。

程序演示必须明确区分：

```text
完整可运行
静态演示版
功能受限演示
视频演示
截图记录
开发中
```

## 第三条：内容优先于动画

重要内容必须存在于真实 DOM 中，而不是只画在 Canvas 里，包括标题、摘要、程序说明、按钮、导航和联系信息。

Canvas/WebGL 负责氛围、场景、粒子、背景和装饰。

即使 JavaScript 被禁用、Canvas 初始化失败、WebGL 不可用、用户开启 Reduced Motion 或素材加载失败，核心内容仍应可读。

## 第四条：滚动动画必须可逆、确定且不劫持输入

- 向下滚动推进；
- 向上滚动自然倒放；
- 停止滚动时停在相应状态；
- 不锁死滚轮；
- 不覆盖浏览器默认滚动；
- 不在回滚时随机重建场景；
- 同一进度产生基本一致的画面。

## 第五条：渐进增强与无障碍是功能

必须支持：

- `prefers-reduced-motion`；
- 键盘操作和可见焦点；
- 合理标题层级；
- 图片替代文本；
- 可读颜色对比；
- 移动端触控；
- Canvas 失败降级；
- 不自动播放声音。

Reduced Motion 模式仍要保留三个可辨识世界，不能退化为单一纯色背景。

## 第六条：只使用原创或授权清晰素材

禁止直接使用或高度仿制任天堂、超级玛丽等游戏原始素材，电影截图、音乐和角色，来源不明像素素材及未授权字体。

允许原创素材、程序化生成素材和许可证允许使用的开源素材，并保留许可记录。

## 第七条：性能预算在设计阶段执行

必须按画质控制粒子、限制 DPR、延迟加载非首屏场景、页面切后台暂停动画、清理监听器和时间线，并为移动端设计独立降级方案。

## 第八条：不在前端泄露秘密

禁止提交 API Key、数据库账号密码、私有 Token、内部服务地址和云存储密钥。

静态前端中任何最终下发到浏览器的值都不能视为秘密。

## 第九条：文档是工程契约

路由、内容模型、静态部署、阶段进度、技术栈、程序演示安全边界和测试命令发生变化时，必须同步更新文档。关键决策写 ADR，不能只存在于聊天记录中。

## 第十条：完成必须有证据

不得在没有证据时宣称“全部完成”“验证通过”“可以部署”。证据包括终端命令、退出码、自动化测试、静态输出目录、浏览器验证或人工验收。

未亲自验证的状态必须写为：

```text
据现有信息推测
已由上一代理汇报
尚未独立验证
```

## 宪章修订流程

修改本宪章必须：

1. 新建 ADR；
2. 说明修改动机；
3. 说明受影响规范和任务；
4. 更新版本号；
5. 更新 `CHANGELOG.md`；
6. 获得项目所有者确认。
