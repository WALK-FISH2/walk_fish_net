# 实施计划 Plan

版本：1.2.2

本计划基于现有第一轮骨架继续推进，不要求无故重建项目。

## 0. 当前状态基线

据 Codex 汇报已经完成三段滚动框架、Markdown/MDX 内容、搜索筛选、详情页、关于我、404、响应式、Reduced Motion、SEO、降级模式、类型检查、ESLint、生产构建和路由测试。这些状态记为“已汇报，待独立验证”。

## 阶段 1：仓库与静态导出审查

目标：明确当前 vinext/React 的真实构建方式，确认是否能输出纯静态目录，确认动态文章和程序路由能预生成，并确认 GitHub Pages 与 Cloudflare Pages 可部署。

输出：静态目录、构建命令、路由清单、静态预览命令、部署说明和 ADR 0002 结论。

退出条件：全部公开路由可脱离 Node 访问；否则提出迁移方案并获得批准。

## 阶段 2：领域语义修正

把原“项目 projects”调整为“做点啥呢 / Programs / 本人程序示例”。

工作：

- 审查现有 projects 模型；
- 新建或迁移 programs；
- 增加演示类型、本人完成内容、限制说明和程序状态；
- 调整导航；
- 兼容 `/projects`；
- 更新详情、搜索、示例内容和测试。

## 阶段 3：陆地世界美术升级

完成天空、两层云、远山、丘陵、草地、前景、原创引导物、路牌式文章入口、多层视差和移动端降级。

退出条件：不看文字也能判断为陆地，文章入口融入世界，反向滚动稳定。

## 阶段 4：陆地下潜与深海世界

完成水位连续上涨、短暂折射、下潜、深海渐变、光束、鱼群、气泡、海草、珊瑚、水母、海床和海底程序档案。

退出条件：陆地到深海不是直接切色，程序卡片不再像普通企业卡片。

## 阶段 5：海洋到星空核心过渡

```text
0.64–0.66 深海尾段预热，固定粒子池开始增密
0.66–0.69 三股桌面/两股移动端定向气泡流形成
0.69–0.72 气泡增亮，深海颜色连续向夜蓝过渡
0.72–0.755 同一 MorphParticle 的气泡外圈收缩为像素星点
0.755–0.785 少量既有星点沿固定像素轨迹变为流星
0.785–0.80 银河、星空与 About 内容稳定落定
```

M4.5 负责陆海三层浪与泡沫幕；M5 不再重复绘制海浪，而是复用一个固定种子的统一粒子池完成气泡、星点和流星三态插值，并让水下光束连续成为像素银河。背景与 DOM 全程保持水平，支持按绝对进度反向倒放、375px 降级和 Reduced Motion 静态转换。背景世界旋转已作为需求更正纳入 M4.5 并全面取消，M5 不得重新引入。

## 阶段 6：星空世界与动效模式入口

三层星点、星云、银河、流星、星座、漂浮信号站、关于我、文章与程序星座入口、页脚和重启旅程已经完成并有既有证据。

M6 新增“动效模式入口”收尾范围：普通网址首次默认完整动画；固定 UI 提供完整/简化动画开关；用户主动选择在本地持久化；`?motion=full` 与 `?motion=reduce` 保留为不持久化的临时覆盖；系统 `prefers-reduced-motion: reduce` 继续被识别并显示简化建议，但不再自动取代产品默认模式。该增量必须复用现有完整动画和 Reduced Motion 分支，不重新实现 M3～M5.5，也不改变章节区间、场景高度、路由或内容系统。

M6 初次实施后，项目所有者于 2026-07-22 发现两项生产回归，因此 M6 重新打开，暂不进入 M7：

1. 完整动画切换为简化动画后，偶发出现物理滚动已到底、进度仍停在深海到星空之间的状态；
2. 产品默认改为完整动画后，星空左侧原本常显的四行“星星 + 文章/Program 标题”退化为只有 hover/focus 才显示标题，破坏了已确认的历史效果。

修复范围只包括：统一模式切换后的滚动范围与进度恢复、恢复四项常显星座入口、补齐重复切换和关键进度验收。不得重做 M5/M5.5 星空背景、行星、流星、海空过渡，不得修改文章/Programs 数据模型、路由或首页其他视觉系统。项目所有者确认后已完成 M6-16～M6-19；七点双向矩阵、5 轮快速切换、桌面/375px、简化动画、Canvas fallback、键盘/链接和生产构建均通过，M6 正式关闭。

## 阶段 6.1：深海 Programs 左右分栏与标题渐隐

M6.1 是 M6 完成后的独立内容层收尾，不重新实现 M4～M6。桌面完整动画把 Programs 区域组织为左右双轨：“沉在海里的程序档案”标题在 section 左上安全区停驻；三张档案卡片整体向右移动，但继续沿用当前完整纵向排列、现有间距和自然出场，不新增单卡 sticky、独立步骤或逐张接棒系统。

最后一张卡片划过 Programs 尾段时，标题提前从深海亮色变暗并降低透明度，在卡片离开主要阅读区时完全消失，形成“隐入夜空”的连续效果。标题退场可反向恢复，不旋转、不闪烁、不修改 M5 海空过渡。

移动端竖屏和简化动画继续使用当前稳定的单列普通文档流：标题在上，三张卡片在下。中间宽度不足时同样回退到单列。阶段不修改 Program 内容模型、路由、深海 Canvas、美术素材、M5 边界或世界 `0°` 约束。

退出条件：桌面标题/卡片和卡片/卡片矩形相交为 0，三张卡片保持现有完整排列，标题在最后一张卡片划过时完成可逆的渐暗消失；375px 顺序与现状一致；正向、反向、完整/简化切换、键盘、Canvas fallback 和静态构建均有证据。详细要求见 `docs/product/m6-1-underwater-programs-layout-spec.md`。

## 阶段 7：程序演示系统

完成程序注册表、静态组件、iframe、外部链接、视频/GIF/截图、加载状态、错误边界、沙箱、限制提示、懒加载，并至少接入一个真实本人程序。

## 阶段 8：内容替换与个人化

替换姓名、介绍、真实文章、真实程序、联系方式、封面和占位素材；检查个人信息和素材许可。

## 阶段 9：质量与发布

完成类型、Lint、单元、E2E、路由刷新、静态预览、性能、Reduced Motion、键盘、移动端、SEO、404、部署和最终文档。

## 实施原则

每一阶段：先更新任务，再实施，运行验证，更新文档，输出风险，然后进入下一阶段。不得把阶段 3～7 合并成一次不可审查的大改动。
