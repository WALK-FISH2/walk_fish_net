# 给 Codex 的 M5.5 执行提示词

请基于已经完成的 M5，实现一次独立的 M5.5 视觉抛光。不要重新实现 M5，也不要扩大为全站重构。

开始编码前，请先阅读：

- `docs/product/m5-5-star-ocean-polish-spec.md`
- `docs/workflows/m5-5-starry-sky-polish-task.md`
- `docs/workflows/m5-5-wave-bubble-polish-task.md`
- `assets/reference/m5-5/` 下的现状截图和六张参考素材

## 一、先检查现有实现

请先定位并说明：

1. M5 星空 section、滚动容器和视差层分别在哪里；
2. 固定流星当前由 DOM、SVG、Canvas 还是贴图实现；
3. 右上行星及光环当前如何分层；
4. 三层浪花当前使用哪些组件、样式和素材；
5. 哪些 transform 由 scroll progress 驱动。

确认现状后再修改，不要在未理解现有结构时新建重复系统。

## 二、M5.5-A 星空抛光

### 1. 固定流星

- 保留现有固定流星的数量、位置和原视差关系。
- 将方向统一修正为左上到右下。
- 确保拖尾位于流星头的左上侧。

### 2. 随机高速流星

新增一个独立于滚动进度的动态流星系统：

- 只在星空 section 可见时运行；
- 任意时刻最多 1~2 颗；
- 出现间隔随机，使用项目所有者实际试用后确认的 1~3 秒配置；
- 单颗持续约 300~900ms；
- 总体朝右下，角度允许轻微随机；
- 流星头更亮，尾迹较短并快速淡出；
- 不读取 scroll progress 计算运动位置；
- 不放在会被滚动 transform 的背景容器中；
- 离开 section、页面隐藏或组件卸载时暂停和清理；
- 防止重复进入章节后叠加多个 RAF/timer；
- reduced-motion 时关闭随机流星。

优先采用独立 Canvas + `requestAnimationFrame`。如果现有架构更适合 DOM，也可以使用 DOM，但必须满足以上生命周期和滚动解耦要求。

### 3. 星体/行星光环

重做右上行星和主要星体的光环：

- 主体核心；
- 内侧亮层；
- 更大的外侧扩散层；
- 行星环/轮廓环；
- 必要时增加非常轻的环境散射。

层与层之间要有明显尺寸、透明度和亮度差。不要复制同一张圆图只改透明度，也不要用大范围 blur 破坏像素风。行星环要处理前后遮挡关系。

## 三、M5.5-B 浪花与气泡抛光

- 保留现有三层结构和原滚动区间。
- 使用 `assets/reference/m5-5/` 中的素材确定新版轮廓和泡沫语言。
- 采用“处理后的透明 sprite + 轻量代码运动”，不使用纯算法临时生成浪花轮廓。
- 前景层最大、卷曲最强、泡沫最多；中景层较平缓；背景层最低最轻。
- 三层使用不同速度、相位和透明度，避免复制感。
- 横向气泡带用于海面泡沫；竖向气泡簇用于海水内部，不得机械铺满海面。
- 将参考图处理为透明生产素材，去除黑底和多余空白，保留清晰大像素。
- 处理横向接缝和不同视口裁切。
- 不改变 section 高度、进度条语义和 M5 章节节奏。

## 四、工程要求

- 关键参数集中配置，不要散落魔法数字。
- 不修改与 M5.5 无关的文案、导航和组件。
- 新增动画需要完整 cleanup。
- 检查桌面端、移动端和 reduced-motion。
- 不得产生控制台错误、重复循环或明显性能下降。

## 五、完成后的汇报格式

完成后请提供：

1. 修改和新增的文件清单；
2. 固定流星方向如何修正；
3. 动态流星如何与 scroll progress 解耦；
4. 动态流星的激活、暂停和 cleanup 逻辑；
5. 光环的最终分层结构；
6. 三层浪花分别使用的生产素材和参数；
7. 是否改变了 M5 原有滚动区间；
8. 已完成的桌面端、移动端与 reduced-motion 验证；
9. 仍需人工确认的视觉细节。

## 六、完成标准

只有同时满足以下条件才算完成 M5.5：

- M5 基线功能和章节节奏没有回归；
- 固定流星全部朝右下；
- 动态流星少量、偶发、快速，并且不随滚动层移动；
- 动态流星只在星空区运行且生命周期清理正确；
- 行星光环具有清楚的多层关系；
- 三层浪花和气泡明显比 M5 基线更自然；
- 不存在明显接缝、遮挡、布局抖动和性能问题。
