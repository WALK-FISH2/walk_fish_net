# 测试策略 Testing Strategy

## 1. 测试层级

### 静态检查

TypeScript、ESLint、内容 schema、Markdown 链接和资源引用。

### 单元测试

`localProgress`、阶段判定、画质选择、Reduced Motion、内容排序、Program schema、DemoRegistry 和旧路由映射。

### 集成测试

文章加载、程序加载、动态详情、演示容器、错误边界和静态路由生成。

### E2E

首页滚动、反向滚动、节点导航、文章搜索、程序筛选、程序演示、详情刷新、404、移动端、键盘和 Reduced Motion。

## 3. 静态输出测试

构建后用纯静态服务器预览，关闭开发服务器，直接输入详情 URL 并刷新，检查资源和 404。

## 4. 演示隔离测试

验证 iframe 不能访问主站敏感状态、不能顶层跳转、样式不污染主站、失败时可关闭、移动端可滚动和加载超时有提示。

## 5. 完成证据

记录日期、工作树状态、命令、退出码、浏览器、视口、Reduced Motion 和失败截图。
