# 测试策略 Testing Strategy

## 1. 测试层级

### 静态检查

TypeScript、ESLint、内容 schema、Markdown 链接和资源引用。

### 单元测试

`localProgress`、阶段判定、画质选择、动效模式优先级、Reduced Motion、偏好持久化与存储失败降级、内容排序、Program schema、DemoRegistry 和旧路由映射。

### 集成测试

文章加载、程序加载、动态详情、演示容器、错误边界和静态路由生成。

### E2E

首页滚动、反向滚动、节点导航、文章搜索、程序筛选、程序演示、详情刷新、404、移动端、键盘和动效模式。M6 动效模式至少覆盖：普通网址首次完整、手动切换、刷新保持、站内跳转保持、`?motion=full`/`?motion=reduce` 临时覆盖且不污染保存值、系统 Reduced Motion 建议、存储不可用、Canvas fallback，以及连续切换后无重复 RAF/timer/监听器。

M6 回归矩阵必须在 64%、72%、76%、79%、80%、82%、90% 执行 full→reduce 与 reduce→full。每次记录切换前后 progress、ScrollTrigger start/end、scrollY、scrollMax、phase，并继续向下到 100%、再向上回到 0%。硬性断言为：进度误差 `<= 0.01`；物理底部时 progress `>= 0.999` 且 phase 为 space；连续 5 轮切换后只有一个有效恢复事务、一个 ScrollTrigger、一个 MeteorOverlay 和预期 Canvas 数量。

星空视觉/交互回归同时断言：四个 `.constellation-star` 均存在、标题计算后可见且 opacity 为 1；完整/简化模式都采用桌面左侧纵向列表；375px 进入文档流且无横向溢出；四个链接可点击、可 Tab 聚焦并到达两篇文章与两个 Program 的正确路由。截图至少包括完整模式 90%、简化模式 90% 和 375px 90%。

2026-07-22 M6 最终执行结果：七点双向矩阵最大误差 `0.0002`；每点均完成 100%/0% 往返；5 轮快速切换误差 0；桌面、375px、系统 Reduced Motion 建议、完整/简化、Canvas fallback、阶段节点、Program 链接焦点/导航和新建标签页控制台通过。自动化测试为 14/14，生产构建生成 15 个静态 HTML。

M6.1 必须增加独立的 Programs 内容层回归矩阵。桌面在 43%、46%、51%、56%、59%、61%、64% 记录标题、三张卡片、浮动导航、阶段导航和动效开关的 `getBoundingClientRect()`；硬性断言为标题/卡片、卡片/卡片及标题/fixed UI 的相交面积均为 0，当前主阅读卡片不得被 fixed UI 遮挡。普通流卡片已经离开主要阅读区的边缘片段可以从浮动导航下方自然经过，但不得盖住导航或拦截其交互。三张卡片必须始终位于同一个普通文档流容器，计算样式不得为 sticky/fixed，也不得新增逐卡步骤或接棒显隐状态。

标题退场另记录计算后的颜色、opacity 和 translateY：进入最后一张卡片尾段后 opacity 必须单调下降，在 Programs 退出时到达 `0`；从 61% 反向回到 43% 时颜色与 opacity 连续恢复。全过程不得出现第二个 ScrollTrigger、页面级 pin 或滚动锁。

M6.1 响应式矩阵至少覆盖 1280×720、1920×1080 和 375×812。375px 必须断言 DOM 与视觉顺序均为标题 → 卡片 01 → 卡片 02 → 卡片 03 → 总入口，所有元素为普通文档流且 `scrollWidth <= clientWidth`。完整/简化切换要在 Programs 中段执行，切换后误差不超过 `0.01`，并继续验证 100%/0% 往返。Canvas fallback、键盘 Tab、真实 Program 链接和控制台同时纳入复验。

## 3. 静态输出测试

构建后用纯静态服务器预览，关闭开发服务器，直接输入详情 URL 并刷新，检查资源和 404。

## 4. 演示隔离测试

验证 iframe 不能访问主站敏感状态、不能顶层跳转、样式不污染主站、失败时可关闭、移动端可滚动和加载超时有提示。

## 5. 完成证据

记录日期、工作树状态、命令、退出码、浏览器、视口、系统 Reduced Motion、最终解析模式、偏好来源（URL/本地/默认）、切换前后活动循环数量和失败截图。
