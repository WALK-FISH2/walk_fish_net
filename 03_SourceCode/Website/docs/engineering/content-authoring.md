# 内容维护 Content Authoring

## 添加文章

1. 新建 Markdown/MDX；
2. 填写 Frontmatter；
3. 添加封面；
4. 本地预览；
5. 检查链接；
6. 运行内容校验；
7. 构建并发布。

## 添加程序

1. 确认是本人编写或贡献明确；
2. 创建 Program 内容文件；
3. 写清 `ownerContribution`；
4. 选择 `demoType`；
5. 写清 `limitations` 和隐私；
6. 添加截图；
7. 注册演示；
8. 测试移动端和失败情况。

## 演示文案模板

```text
演示状态：静态演示版

该页面用于展示主要交互和界面流程。
以下功能未接入：登录、数据库、云同步和正式后端接口。
演示数据仅保存在当前浏览器，刷新或清除存储后可能丢失。
```

## 程序详情建议结构

```text
程序简介
为什么写它
核心功能
本人完成内容
技术方案
关键难点
程序演示
当前限制
隐私说明
后续计划
源码或外部链接
```

## 发布检查

无占位姓名、无虚假功能、无内部截图、无秘密、无无效链接、图片有 alt、程序状态和日期正确。
