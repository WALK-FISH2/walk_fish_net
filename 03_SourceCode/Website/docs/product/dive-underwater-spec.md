# 下潜与深海场景规格

状态：待实施  
对应任务：M4  
适用项目：Pixel//Walk 个人网站

---

## 1. 本阶段目标

建立从明亮陆地连续进入深海的完整体验。

本阶段不是简单把陆地背景切换为深蓝色，而是要让访问者清楚感受到：

```text
陆地开始被水覆盖
→ 水位不断上涨
→ 视角进入水下
→ 水面逐渐远离
→ 镜头向海底下潜
→ 进入程序档案所在的深海世界
```

所有关键过程必须由滚动进度控制，并支持反向倒放。

本阶段中的“做点啥呢”板块专门展示站点所有者本人编写的程序示例、工具、网页应用、交互实验和程序演示，不是泛化的项目经历列表。

---

## 2. 阶段范围

本阶段对应首页全局滚动进度：

```text
0.30–0.38  陆地到深海的下潜过渡
0.38–0.66  深海程序档案世界
```

建议下潜过渡时间线：

```text
0.300–0.315  陆地环境开始变化
0.315–0.330  屏幕底部出现水面
0.330–0.345  水位上涨到屏幕中部
0.345–0.360  水线经过内容与场景
0.360–0.370  水覆盖全屏，进入水下
0.370–0.380  镜头继续下潜，水面退至顶部
```

建议深海内部节奏：

```text
0.380–0.430  浅层海水，水面仍隐约可见
0.430–0.500  深海环境建立
0.500–0.590  Programs 程序档案主要展示区
0.590–0.640  深海尾段，水流逐渐增强
0.640–0.660  为海洋到星空过渡做准备
```

所有区间应集中放入统一配置文件，不得散落在多个组件内。

## 3. 实现原则

### 3.1 可逆

必须满足：

- 向下滚动时，水位上涨并进入深海；
- 向上滚动时，水位下降并返回陆地；
- 回滚过程中不重置鱼群、气泡和场景；
- 水线不能瞬移；
- 颜色不能闪烁；
- DOM 内容和场景阶段保持一致；
- 用户停止滚动后，叙事动画停留在当前进度。

环境循环动画可以继续缓慢运行，但不能改变叙事状态。

### 3.2 单一进度来源

首页只能存在一个主滚动进度来源：

```ts
type StoryProgress = number; // 0..1
```

每个场景使用局部映射：

```ts
function localProgress(
  globalProgress: number,
  start: number,
  end: number,
): number {
  return Math.min(
    1,
    Math.max(0, (globalProgress - start) / (end - start)),
  );
}
```

禁止：

- 多个互相冲突的全局 scroll 监听；
- 只使用 IntersectionObserver 播放一次性动画；
- 用户向上滚动时重新播放另一套不一致动画；
- 使用全屏视频替代场景。

### 3.3 Canvas 与 DOM 分工

Canvas 或 WebGL 负责：

- 水面；
- 海水；
- 光束；
- 鱼群；
- 气泡；
- 海草；
- 珊瑚；
- 岩石；
- 水母；
- 海床；
- 粒子；
- 视差；
- 折射装饰。

DOM 负责：

- Programs 标题；
- 程序名称；
- 程序简介；
- 技术栈；
- 程序状态；
- 按钮；
- 演示说明；
- 导航；
- 无障碍文本。

不得把重要文字只绘制在 Canvas 中。

## 4. 陆地环境变化

在水面出现之前，陆地环境需要给出自然预兆。

### 4.1 天空

随着全局进度从 `0.300` 到 `0.315`：

- 天空亮度轻微下降；
- 天空色温略微变冷；
- 云层移动速度提高约 10%～20%；
- 太阳或高光亮度轻微降低；
- 不要突然切换成深色天空。

### 4.2 陆地元素

- 远山对比度略微降低；
- 草地颜色逐渐偏冷；
- 前景草叶轻微向同一方向摆动；
- 原创角色或视觉引导物可以出现观察水面的动作；
- 不新增一次性不可逆动画。

### 4.3 过渡提示

可以加入：

- 少量水汽粒子；
- 地面反光；
- 远处水声图标；
- 风向变化。

这些只用于暗示，不应抢夺主要内容注意力。

## 5. 水面规格

### 5.1 水面结构

水面至少包含：

1. 远层波纹；
2. 主水线；
3. 近层泡沫或反光；
4. 少量水雾或亮点。

水面不能只是一个平直矩形。

必须具有：

- 像素化波峰；
- 像素化波谷；
- 局部泡沫；
- 不同速度的多层波动；
- 横向连续性；
- 在不同分辨率下不会断裂。

### 5.2 水面颜色

推荐：

```css
--water-line-light: #b9f7f1;
--water-line-mid: #64d8dc;
--water-line-dark: #1d91a9;
--water-foam: #e8ffff;
--water-reflection: rgba(255, 255, 255, 0.35);
```

水面应与陆地天空区分，但不能突兀。

### 5.3 水位运动

水位必须直接由局部滚动进度控制。

建议：

```ts
const waterRise = localProgress(globalProgress, 0.315, 0.370);
```

运动要求：

- 起始时从屏幕底部以下进入；
- 逐渐上涨；
- 经过屏幕中央；
- 最终覆盖整个视口；
- 不能依赖固定播放时长；
- 用户快速滚动时不能丢失阶段；
- 用户缓慢滚动时不能自行播放完。

### 5.4 水线波动

建议：

- 主体位置由滚动决定；
- 波纹细节由时间循环决定；
- 波纹振幅随画质等级调整；
- 移动端减少层数；
- Reduced Motion 使用静态或极轻微波纹。

---

## 6. 水线经过内容时的折射

### 6.1 目标

当水线经过 DOM 内容时，产生短暂的“进入水下”感觉。

### 6.2 范围

折射只作用于水线上下约：

```text
桌面端：40px～80px
移动端：24px～48px
```

不得让整个页面持续扭曲。

### 6.3 表现方式

可以选择：

- 像素水平错位；
- 小范围波形位移；
- 色散边缘；
- 轻微上下偏移；
- 透明度和色温变化。

禁止：

- 大范围模糊文字；
- 让按钮无法点击；
- 改变 DOM 布局；
- 高频闪烁；
- 强烈色散导致不可读。

### 6.4 技术边界

推荐采用：

- CSS mask；
- 小范围 overlay；
- Canvas 上的折射装饰；
- 单独的伪元素。

不推荐对整块 DOM 使用高成本实时 Shader。

### 6.5 Reduced Motion

Reduced Motion 下：

- 取消动态折射；
- 仅使用短暂颜色变化或像素遮罩；
- 保持文字始终清晰。

## 7. 全屏入水

当进度进入 `0.360–0.370`：

- 水面覆盖整个屏幕；
- 背景色进入水下青蓝；
- 开始出现气泡；
- 陆地整体对比度降低；
- 光线发生水下色偏；
- 画面顶部仍可以短暂看到水面高光。

不得直接销毁陆地场景后切换深海。

陆地元素应逐步：

```text
清晰
→ 色偏
→ 低对比
→ 剪影
→ 消失
```

---

## 8. 镜头下潜

### 8.1 下潜感

在 `0.370–0.380`：

- 水面向屏幕顶部退去；
- 陆地剪影远离；
- 气泡向上运动；
- 背景逐渐加深；
- 光束从顶部进入；
- 近景元素相对上移，形成镜头向下的错觉。

### 8.2 水面远离

水面不应在入水后立刻消失。

应在屏幕顶部保留短时间：

- 高光；
- 波纹；
- 明亮轮廓；
- 少量泡沫。

随着深度增加逐渐淡出。

### 8.3 深度提示

可以显示轻量深度文本，例如：

```text
DEPTH 120
DEPTH 680
DEPTH 1800
SIGNAL FOUND
```

要求：

- 只作为装饰；
- 不遮挡内容；
- 与滚动进度对应；
- 不虚构真实测量精度。

---

## 9. 深海场景层级

深海至少包含以下层级。

### 9.1 远景背景

包括：

- 从浅蓝到藏青的纵向渐变；
- 水面方向透下来的微弱光束；
- 远处遗迹轮廓；
- 远层鱼群；
- 稀疏漂浮粒子。

建议视差速度：

```text
0.10–0.20
```

### 9.2 中景主体

包括：

- 较清晰的鱼群；
- 发光水母；
- 海底岩石；
- 程序档案设备；
- 探测器；
- 中层气泡。

建议视差速度：

```text
0.30–0.50
```

### 9.3 近景环境

包括：

- 海草；
- 珊瑚；
- 较大气泡；
- 发光粒子；
- 小型海洋生物剪影。

建议视差速度：

```text
0.60–0.80
```

### 9.4 前景遮挡

包括：

- 大型海草剪影；
- 岩石边缘；
- 偶尔经过屏幕的近距离气泡；
- 局部珊瑚枝条。

建议视差速度：

```text
0.90–1.20
```

前景不能持续遮挡正文和按钮。

## 10. 深海颜色

深海不能继续使用陆地阶段的亮青蓝色作为主背景。

推荐变量：

```css
--sea-surface: #176b8f;
--sea-upper: #0b5275;
--sea-mid: #092e55;
--sea-lower: #061f3d;
--sea-floor: #04162f;

--sea-glow: #78eadb;
--sea-cyan: #25bec4;
--sea-light: #b6fff2;
--sea-coral: #ff806e;

--sea-panel: #071f3d;
--sea-panel-soft: #0b2c50;
--sea-text: #e5faff;
--sea-muted: #8db9cb;
```

背景应从顶部到底部逐渐变暗。

程序内容区域必须保持足够对比度，不能为了“深邃”降低可读性。

---

## 11. 光束

### 11.1 来源

光束从画面顶部水面方向透下。

### 11.2 表现

- 至少两条不同宽度；
- 不完全平行；
- 缓慢漂移；
- 顶部较亮，底部逐渐衰减；
- 不遮挡程序卡片；
- 不做成现代霓虹灯。

### 11.3 性能

高画质：

- 多条半透明光束；
- 轻微噪声；
- 小范围动态。

中低画质：

- 使用简单渐变多边形；
- 减少数量；
- 降低更新频率。

---

## 12. 气泡系统

### 12.1 对象池

必须使用对象池或复用机制。

禁止每帧不断创建和销毁气泡对象。

### 12.2 属性

每个气泡可以包含：

```ts
type Bubble = {
  x: number;
  y: number;
  radius: number;
  speed: number;
  drift: number;
  depthLayer: number;
  phase: number;
};
```

### 12.3 运动

- 主要向上；
- 有轻微横向漂移；
- 近景较快；
- 远景较慢；
- 大小和速度有差异；
- 不同步移动；
- 不全部从同一位置生成。

### 12.4 回滚

气泡的叙事状态不能因为向上滚动突然重置。

可以：

- 环境循环继续；
- 透明度随场景进度调整；
- 使用固定种子初始化。

---

## 13. 鱼群

至少实现两层。

### 13.1 远层鱼群

- 尺寸较小；
- 对比度低；
- 数量较多；
- 速度较慢；
- 可使用剪影。

### 13.2 中层鱼群

- 尺寸更明显；
- 数量较少；
- 速度略快；
- 偶尔改变队形；
- 不穿过主要文字区域。

### 13.3 移动方向

不同鱼群可以从不同方向通过，但不能频繁逆向造成混乱。

### 13.4 移动端

- 减少数量；
- 降低动画频率；
- 避免占用主要触控区域。

---

## 14. 发光水母

### 14.1 数量

- 桌面端少量；
- 移动端进一步减少；
- 不作为主要视觉焦点。

### 14.2 动画

- 缓慢上下浮动；
- 轻微脉动；
- 发光强度变化；
- 不使用高频闪烁。

### 14.3 位置

- 远景或中景；
- 避开程序卡片标题和按钮；
- 可以在卡片之间形成视觉引导。

---

## 15. 海草、珊瑚、岩石与海床

### 15.1 海草

- 不同高度；
- 不同相位摆动；
- 水流方向基本一致；
- 不能整组完全同步；
- 靠近 `0.640–0.660` 时摆动逐渐增强，为下一过渡预热。

### 15.2 珊瑚

- 使用原创像素形态；
- 提供少量暖色点缀；
- 不宜过度鲜艳；
- 不遮挡内容。

### 15.3 岩石

- 形成海底层次；
- 可以承载程序终端；
- 避免规则几何块直接堆叠。

### 15.4 海床

必须清晰可见。

包括：

- 地形起伏；
- 岩石；
- 沙地；
- 海草根部；
- 程序档案装置的落点。

不能让深海背景像没有底部的纯色空间。

## 16. Programs 程序档案

### 16.1 领域定义

“做点啥呢”展示的是站点所有者本人编写的程序。

每个条目应明确：

- 程序名称；
- 解决的问题；
- 本人完成内容；
- 程序状态；
- 技术栈；
- 演示方式；
- 当前限制；
- 隐私与数据行为。

### 16.2 视觉容器

程序条目应融入深海世界，可以设计成：

- 海底数据终端；
- 沉没档案舱；
- 探测器屏幕；
- 舷窗式程序窗口；
- 发光信号设备；
- 遗迹中的程序记录。

禁止：

- 所有程序都使用完全相同的普通大矩形；
- 只通过换边框颜色区分；
- 使用现代玻璃拟态作为主要视觉；
- 把程序卡片悬浮在没有环境关联的纯背景上。

### 16.3 DOM 内容

每张程序卡片至少显示：

- 程序名称；
- 一句话说明；
- 程序状态；
- 技术栈；
- 演示类型；
- 查看详情；
- 立即体验；
- 静态演示限制。

所有文字和按钮必须使用真实 DOM。

### 16.4 出现方式

建议：

- 随镜头下潜进入视野；
- 从远处逐渐浮近；
- 从海底遗迹中亮起；
- 被光束扫描后显示。

禁止只使用简单 opacity 淡入作为唯一效果。

### 16.5 卡片布局

- 桌面端可以左右交错；
- 不要所有卡片都居中堆叠；
- 卡片之间应留出环境空间；
- 不遮挡鱼群和海草的主要层次；
- 移动端改为单列。

## 17. 程序演示状态

Programs 卡片和详情页需要支持：

```text
完整可运行
静态演示版
功能受限演示
视频演示
截图记录
开发中
```

对于静态演示版，显示：

```text
这是静态演示版。
部分后端、数据库、登录、云同步或实时功能未接入。
```

不得通过伪造成功接口让访客误以为功能完整上线。

## 18. 深海尾段

当进度接近 `0.640–0.660`：

- 海草摆动增强；
- 气泡速度略微提高；
- 鱼群逐渐离开；
- 水流粒子方向更加一致；
- 光束轻微倾斜；
- 程序卡片开始退出主要视野；
- 为后续海洋到星空过渡预热。

本阶段不实现最终浪花翻转，只提供自然预兆。

---

## 19. 移动端

移动端必须保留：

- 水面出现；
- 水位上涨；
- 入水；
- 下潜；
- 深海环境；
- Programs 程序档案；
- 场景差异。

移动端调整：

- 减少鱼群数量；
- 减少水母；
- 减少前景气泡；
- 简化水线折射；
- 降低视差强度；
- 程序卡片单列；
- 不使用大幅横向位移；
- 前景元素不能遮挡按钮；
- 不出现横向滚动条。

---

## 20. Reduced Motion

Reduced Motion 模式下：

- 水位上涨改为短像素遮罩；
- 不执行大幅镜头移动；
- 不使用文字折射；
- 不使用快速视差；
- 保留静态水面；
- 保留深海渐变；
- 保留鱼群剪影；
- 保留海草和海床；
- 保留 Programs 程序终端；
- 三个世界仍然清晰可辨。

Reduced Motion 不能退化为单一纯蓝背景。

## 21. 画质等级

### High

- 完整水面层；
- 局部折射；
- 多条光束；
- 两层鱼群；
- 完整气泡池；
- 水母；
- 多层海草；
- 前景遮挡。

### Medium

- 减少气泡；
- 简化折射；
- 减少鱼群；
- 减少水母；
- 光束简化。

### Low

- 单层水面；
- 无实时折射；
- 少量气泡；
- 少量鱼群；
- 静态或轻微海草；
- 保留海床和程序终端。

### Static

- Reduced Motion；
- Canvas 失败；
- 极低性能设备；
- 使用静态背景与像素遮罩。

## 22. 性能要求

- DPR 上限为 2；
- 页面隐藏时暂停环境动画；
- 气泡使用对象池；
- 鱼群使用精灵复用；
- 不每帧创建大量数组和对象；
- 不使用全屏视频；
- 不把全部程序演示打包进首页；
- 场景资源按接近阶段时加载；
- 场景销毁时清理 Ticker、监听器、Observer 和纹理引用；
- 移动端粒子数量应明显低于桌面端。

目标：

```text
普通桌面设备：尽量接近 60fps
中低端移动设备：维持可接受的 30fps
Reduced Motion：接近普通静态页面成本
```
