# M6.2 跨世界小章鱼主题角色规格

状态：已完成实现与正式验收
日期：2026-07-23

## 1. 背景与目标

M6.2 已将原陆地程序化像素旅行者统一替换为网站主题形象“小章鱼”，并通过普通、潜水和宇航员三种形态串联陆地、海洋与星空。

M6.2 是 M6 完成后的独立角色视觉与交互增量，不重新实现 M3～M5.5，不改变现有章节高度、海浪、粒子过渡、Programs 内容系统、路由或文章系统。

## 2. 参考素材与生产素材

参考源位于 `assets/reference/m6-2/`：

| 参考文件 | 角色形态 | 使用阶段 |
| --- | --- | --- |
| `octopus1.png` | 普通小章鱼 | 陆地、陆地下潜前半段 |
| `octopus2.png` | 潜水小章鱼 | 入水后、深海、深海尾段 |
| `octopus3.png` | 宇航员小章鱼 | 星空入场与星空漂浮 |

参考文件不能直接作为未经处理的运行时素材。实施时必须：

- 检查真实 alpha；移除黑底、白底、棋盘格底和多余空白；
- 裁切到角色有效边界，统一三种形态的视觉中心、脚底/触手基线和感知尺寸；
- 保留大像素、硬轮廓和原形象，不重新设计角色；
- 使用 nearest-neighbor，禁止平滑缩放、大范围 blur 或矢量重绘；
- 输出到源码生产素材目录，并记录来源、处理方式和尺寸；
- `assets/reference/` 只保留为设计输入，不由生产页面直接读取。

## 3. 不变范围

M6.2 不得修改：

- `0.00–0.30` 陆地、`0.30–0.38` 下潜、`0.38–0.66` 深海、`0.66–0.80` 海空过渡、`0.80–1.00` 星空的全局章节边界；
- 陆地现有横向行程语义、地面线与文章路牌安全走廊；
- M4.5 三层浪花、泡沫带与遮缝节奏；
- M5 固定 `MorphParticle` 池及海洋到星空的颜色、光束和粒子形态变化；
- M5.5 固定/动态流星、右上行星与光环；
- 背景世界固定 `0°` 的约束；
- Programs 字段、内容、卡片布局、路由和文章系统；
- 首页 section 高度、阶段导航和动效模式优先级。

## 4. 单一角色状态模型

三种小章鱼共享一个集中配置和一个角色状态模型，不允许陆地、海洋与星空组件各自保存未说明的时间范围、尺寸、透明度或位移魔法数字。

角色状态至少包含：

```text
form: land | diver | astronaut | hidden
position: x / y
opacity
brightness 或 tint
scale
facing / velocity
bob
entryState
visibilityState
```

叙事位置、形态切换、亮度和可见度由 `globalProgress` 或它派生的纯函数决定；待机浮动与星空 DVD 式漂浮属于独立的轻量时间行为，不得反向修改 `globalProgress`。

## 5. 正向滚动叙事

### 5.1 陆地：普通小章鱼

- 使用 `octopus1.png` 的生产版本替换当前小人；
- 保留当前陆地横向行程与地面安全走廊，初始横向参数继续以 `start: 0.13`、`travel: 0.46` 为基线；
- 完整动画下，停止滚动时仍有与现状接近的轻微上下浮动；
- 小章鱼必须位于 Hero、文章路牌和正文后方，不能拦截链接或按钮；
- 正向和反向滚动时不得在路牌间跳位。

### 5.2 陆地进入海洋

建议沿用现有浪花关键区间：

| 全局进度 | 角色目标 |
| ---: | --- |
| `0.300–0.328` | 普通小章鱼向上跃起，在远浪出现期间到达跳跃高点 |
| `0.328–0.355` | 小章鱼向现有中浪/前浪落下，进入浪花区域 |
| `0.355–0.368` | 由现有泡沫和前浪遮挡角色切换，不在无遮挡画面中直接换图 |
| `0.368–0.380` | 潜水小章鱼建立入水状态，进入浅层海水 |

跳跃必须表现为“先向上、再落入现有浪花”，不得新建第二套浪花，也不得让角色漂在水线之上直接变成潜水形态。向上滚动时按相反顺序恢复：潜水形态退回泡沫遮挡区，再由普通形态跃回陆地。

### 5.3 深海：潜水小章鱼

- 使用 `octopus2.png` 的生产版本；
- `0.380–0.620` 从屏幕右上区域逐渐移动到左下区域；
- 完整动画下叠加轻微上下浮动，浮动不得改变滚动决定的主路径；
- 角色位于 Programs 标题和三张卡片后方，允许被内容短暂遮挡，但不得覆盖或拦截任何交互；
- `0.620–0.660` 向屏幕下方潜出，`0.660` 前完全离开视口；
- `0.660–0.800` 维持角色缺席，让现有 M5 海空形态转换保持清晰。

### 5.4 星空：宇航员小章鱼

- 使用 `octopus3.png` 的生产版本；
- 从海洋完整进入星空时，宇航员小章鱼从屏幕下方升入星空，不能从屏幕上方下降；
- 完成入场后，开始独立于滚动位置的 DVD 待机式漂浮；
- 碰到可用视口边界时改变水平或垂直方向，保持连续速度，不瞬移；
- 完整模式下持续漂浮；页面隐藏、组件卸载、完全离开星空或切换到简化动画时必须暂停并清理；
- 任意时刻只能存在一个宇航员角色、一个活动 RAF 和一套边界反弹状态。

## 6. 星空向上返回海洋的两种分支

宇航员离开星空时不能停止边界反弹。滚动进度只控制它的亮度与可见度，漂浮位置继续由当前 DVD 轨迹推进。

### 6.1 尚未完全消失便重新向下

```text
星空正常漂浮
→ 用户向上进入海空交界
→ 宇航员继续反弹，同时逐渐变暗、降低透明度
→ 尚未完全消失时用户改为向下
→ 不重置位置、不重新入场、不创建新 RAF
→ 在当前反弹轨迹上恢复原始颜色和完全可见
```

该分支不得出现闪现、跳回初始坐标、速度突变或重复角色。

### 6.2 已完全消失后再次向下

```text
宇航员继续反弹并逐渐变暗
→ 可见度到 0，宇航员完全消失并停止/清理漂浮循环
→ 继续向上时，潜水小章鱼在海洋尾段从屏幕下方升起
→ 用户再次向下，潜水小章鱼重新向下潜出
→ 星空重新建立后，宇航员小章鱼再次从屏幕下方升入
→ 入场完成后重新开始单一 DVD 漂浮循环
```

这是一次新的完整入场。宇航员不得从屏幕上方下降，也不得直接在上次消失坐标突然出现。

## 7. 星空亮度与可见度门

宇航员的星空交界状态使用集中配置的可逆门。建议以 `0.800–0.830` 作为初始调校范围：

- `0.800`：宇航员不可见，漂浮循环未运行；
- `0.800–0.830`：从屏幕下方入场，亮度和透明度逐步恢复；
- `>= 0.830`：正常颜色、完全可见并进行 DVD 漂浮；
- 从高进度向上返回 `0.830–0.800` 时，如果宇航员已经进入漂浮状态，则保留当前反弹轨迹，只反向降低亮度和透明度；
- 到达完全不可见边界后才销毁或暂停漂浮循环。

具体阈值允许在浏览器调校时小幅收敛，但不得越过 `0.80` 的星空章节边界，也不得修改 M5 的 `0.660–0.800` 粒子时间线。

## 8. 分层与交互边界

角色属于装饰层，统一使用 `aria-hidden="true"` 和 `pointer-events: none`。

目标层级为：

```text
星空背景与右上行星
< 信号站天线、卫星等装饰
< 宇航员小章鱼
< 信号站文字卡片、星座入口、导航、按钮和 fixed UI
```

因此宇航员可以从文字卡片后方经过，但必须显示在星球和信号站装饰前方。实施时允许拆分“信号站装饰”和“信号站正文”的展示层，但不得复制正文、改变 DOM 阅读顺序或把正文移入 Canvas。

陆地普通小章鱼与潜水小章鱼继续位于场景层中，天然处于 Hero、文章路牌和 Programs 内容后方；入浪阶段必须利用现有浪花/泡沫遮挡完成形态切换。

## 9. 移动端

- 375px 保持普通 → 潜水 → 宇航员的相同叙事顺序；
- 缩小角色尺寸、跳跃高度、海洋斜向行程和 DVD 可用区域；
- 优先避开顶部导航、底部阶段进度、动效模式开关和主要文字按钮；
- Programs 仍使用 M6.1 的标题在上、三张卡片在下布局；
- 不产生页面级横向滚动；
- 触控滚动不得被角色动画监听器阻塞。

## 10. 简化动画与 Canvas 降级

简化动画：

- 关闭持续上下浮动；
- 关闭 DVD 式漂浮和边界反弹；
- 保留三种可辨识的小章鱼形态；
- 使用简化的位置变化、浪花遮挡、形态切换与淡入淡出表达同一叙事；
- 不先启动完整动画再切换为静态状态。

Canvas fallback：

- 核心 DOM、链接和滚动必须继续可用；
- 角色可降级为当前章节对应的静态生产图片；
- 不因 Pixi 初始化失败创建重复的 DOM/Canvas 角色循环；
- 角色始终不拦截键盘、指针或触控。

## 11. 生命周期与性能

- 普通/潜水角色与叙事进度使用现有单一滚动源；
- 宇航员 DVD 漂浮不得读取滚动值计算每帧坐标，但其激活、变暗和清理由星空可见度门控制；
- 切后台暂停，恢复前重新计算视口边界，不能从屏幕外错误反弹；
- resize、动效模式切换、章节往返和组件卸载必须清理事件监听与 RAF；
- 部分变暗后恢复不得启动新的循环；
- 完全消失后重新进入只启动一套新的入场与漂浮循环；
- 不引入大型动画依赖，不增加第二个 ScrollTrigger 或页面级滚动监听器。

## 12. 正式验收

至少验证以下进度与路径：

```text
0%      普通小章鱼替换现有小人
25%     陆地横向行程、待机浮动和人物走廊
30.0%   跳跃开始
32.8%   跳跃高点/浪花接近
35.5%   落入前浪并由泡沫遮挡
36.8%   潜水形态建立
38.0%   浅层海水
50.0%   右上向左下路径与 Programs 遮挡关系
62.0%   深海尾段下潜开始
66.0%   潜水小章鱼完全离场
80.0%   宇航员星空入场门
83.0%   宇航员完全可见并开始/继续 DVD 漂浮
90.0%   星空稳定、边界反弹与层级
```

反向与分支必须分别验证：

```text
90% → 81.5%：宇航员继续反弹，同时变暗但仍可见
81.5% → 90%：同一轨迹恢复颜色与透明度，不重新入场
90% → 80%：宇航员逐渐消失，完全不可见后清理循环
80% → 66% → 62%：潜水小章鱼从下方升起
62% → 66% → 80% → 83%：潜水形态下潜离场，宇航员从下方重新升入并恢复漂浮
38% → 30%：潜水形态经浪花遮挡恢复为普通形态并回到陆地
```

还必须覆盖：

- 1280×720、1920×1080 和 375×812；
- 完整动画与简化动画；
- `prefers-reduced-motion` 建议环境及用户主动切换；
- 页面隐藏/恢复、resize、重复进入星空和连续正反向滚动；
- Canvas fallback；
- 键盘、指针、触控与所有真实链接；
- 浏览器控制台、RAF/监听器数量和明显性能下降；
- Astro Check、ESLint、测试、生产构建和静态路由验证。

## 13. Definition of Done

只有以下条件全部有代码、测试或浏览器证据时，M6.2 才能标记完成：

- [x] 三种生产小章鱼素材完成透明化、裁切、统一基线和来源记录；
- [x] 普通小章鱼替换现有小人，陆地行程与待机浮动无回归；
- [x] 跳跃、入浪遮挡和潜水形态切换连续且可反向；
- [x] 潜水小章鱼完成右上到左下路径及尾段下潜，并可反向升起；
- [x] 宇航员只从屏幕下方升入星空；
- [x] 宇航员持续边界反弹，在部分回滚时一边反弹一边变暗；
- [x] 部分回滚后向下可在同一轨迹恢复；完全消失后向下会重新从下方入场；
- [x] 宇航员位于星球/信号站装饰前、文字卡片和交互内容后；
- [x] 移动端、简化动画、Canvas fallback 和生命周期全部通过；
- [x] M3～M6.1、Programs、路由、静态输出与世界 `0°` 无回归。

验收证据详见 `tasks.md` 的 `M6.2-03`～`M6.2-16` 和 `docs/engineering/adjustment_record.md` 第 11 节。
