# 视觉系统 Visual System

## 1. 风格

核心：二维像素、精细、复古主机感、明确轮廓、环境层次和少量现代光影辅助。

禁止：体素、3D 方块、大量玻璃拟态、全页纯色、所有内容套同一种矩形卡片和直接复制经典游戏素材。

## 5. 字体

像素字体用于 Logo、小标题、按钮、标签、状态和关卡文字；正文使用系统中文字体，不使用低清晰度像素字体写长段中文。

## 6. 像素规则

```css
.pixel-art {
  image-rendering: pixelated;
  image-rendering: crisp-edges;
}
```

精灵使用 nearest-neighbor，DPR 上限 2，精灵尽量落在整数坐标，UI 阴影使用整数像素。背景渐变和光束可以平滑。

## 7. 容器语言

- 文章：路牌、公告板、关卡入口；
- 程序：海底终端、舷窗、探测器、沉没档案舱；
- 关于我：漂浮信号站、太空档案、星座中心。

## 8. 组件状态

所有交互组件需要 default、hover、focus-visible、active、disabled、loading 和 error 状态，不能只设计 hover。

## 9. 素材许可

记录文件名、来源、作者、许可证、修改情况和项目内许可文件位置。

## 10. 场景方向

陆地、深海、海洋到星空和星空场景始终保持水平。M4.5 的浪花只服务陆海交接；M5 海空过渡通过色彩、光束、透明度与统一粒子形态变化完成，不使用 Canvas 世界旋转、翻转或倾斜制造转场。

海空气泡使用青绿色空心像素环与局部高光；形变时外圈连续收缩，高光逐渐成为暖白像素星核。只有固定池中的少量既有星点可以沿固定坐标轨迹延伸出分段矩形尾迹；桌面流星上限比例为 `5.5%`，375px 为 `2.2%`，Reduced Motion 为 `0`。水下三束光带通过像素块的坐标、宽度、颜色与透明度插值成为斜向银河，但光带、粒子和世界容器本身均不旋转。

M5.5 后，固定流星统一朝右下，头部位于轨迹右下端，短拖尾位于左上。偶发高速流星使用独立时间层，桌面最多 2 颗、移动端最多 1 颗，Reduced Motion 关闭。右上行星采用“环境散射 → 外扩散 → 内亮层 → 后环 → 核心主体 → 前环”的顺序；前后环由主体遮挡关系区分，不使用模糊滤镜。

陆海三层浪的生产轮廓分别来自 `wave-back.png`、`wave-mid.png` 和 `wave-front.png`；前景最大且卷曲最强，中景适中，背景最低最轻。`foam-band-front.png` 与 `foam-band-mid.png` 只沿浪尖和接缝横向分布，`bubble-clusters.png` 只位于水体内部。所有素材使用透明背景、nearest-neighbor 和像素对齐，运行时不直接读取 `assets/reference/`。

## 11. 内容层与场景层的遮挡边界

- 陆地 Hero 和三张文章路牌位于 Canvas 之上，但必须共同让出地面人物走廊；路牌本体、错层位移和路牌柱均计入遮挡边界，不能在人物大部分行走过程中覆盖人物；
- `≤980px` 的文章路牌采用横向轨道并隐藏装饰路牌柱，保留三篇文章和键盘焦点顺序，同时避免单列卡片穿过人物地面线；
- Programs 档案终端保留海底终端、探测器和档案舱的不同轮廓，但标题、正文、操作链接及相邻终端不得互相覆盖；
- 标准动效、`?motion=full`、375px 和 Reduced Motion 共享同一内容层阅读顺序，仅场景运动强度和阶段时长允许变化；
- 通过正常文档流和显式间距解决内容碰撞，不以降低文字透明度、隐藏条目或改变 Program 语义规避问题。

## 12. 动效模式控件

- 动效模式开关属于 `fixedUI`，不得放进 Canvas、滚动视差层或随章节消失的内容卡片；
- 控件沿用现有像素 UI 的边框、整数阴影和短标签语言，显示“动画：完整”或“动画：简化”，不新增大面积浮层；
- 完整、简化、系统建议简化、键盘焦点和按下状态必须可通过文字与轮廓辨认，不能只依赖颜色；
- 375px 下控件不得遮挡进度节点、导航、主要按钮或正文，触控区域不小于现有站点按钮标准；
- 模式切换只改变运动强度与时间行为，不改变三个世界的色板、内容层阅读顺序和固定 0° 世界方向。
