# M5.5 星空与海浪视觉抛光需求说明

> 建议项目路径：`docs/product/m5-5-star-ocean-polish-spec.md`
>
> 里程碑定位：M5 已完成后的增量优化阶段。M5.5 不重新实现 M5，只对已完成页面进行视觉修正、资源替换和动效增强。
>
> 实施状态：2026-07-18 已完成并通过工程与浏览器验收。
>
> 参数修订：2026-07-19 经项目所有者实际试用，将动态流星随机生成间隔由初始建议的 5~14 秒调整为 1~3 秒；其余数量、方向、持续时间、生命周期和 Reduced Motion 约束不变。

---

## 1. 里程碑定位

### 1.1 已完成基线

M5 已经完成并作为本次工作的稳定基线。当前已有内容包括：

- 星空场景与滚动叙事；
- 星点、行星、固定流星等背景装饰；
- 海洋到星空的既有过渡；
- 陆地到海洋的三层浪花结构；
- 已存在的滚动进度、视差与章节切换逻辑。

### 1.2 M5.5 的职责

M5.5 只处理以下增量内容：

1. 修正固定背景流星方向；
2. 新增与滚动解耦的随机高速流星；
3. 重做星体/行星光环层次；
4. 使用新参考素材优化三层浪花与气泡；
5. 完成视觉回归与性能检查。

### 1.3 非目标

本阶段不得主动修改：

- M5 的页面章节顺序；
- 原有文案、导航、按钮和信息结构；
- 既有滚动区间和进度条语义；
- 海洋到星空的核心叙事逻辑；
- 与本需求无关的组件结构或全局样式。

除非现有实现阻碍本需求落地，否则不得以“顺便重构”为理由扩大修改范围。

## 2. 总体设计原则

- 延续当前像素风、深蓝紫色调和复古游戏 UI 语言。
- 新增动效应作为点缀，不覆盖主要内容，不影响文字阅读。
- 区分“滚动驱动场景层”和“基于时间运行的瞬时特效层”。
- 优先增量接入，保证 M5 已完成效果不发生非预期变化。
- 动画参数应集中配置，避免散落硬编码。
- 尊重 `prefers-reduced-motion`，并对低性能设备提供降级策略。

## 3. 星空部分

### 3.1 固定背景流星方向修正

#### 现状

背景中已有固定流星/拖尾装饰，当前视觉方向偏向右上。

#### 需求

- 保留现有固定流星，不删除、不改成全部动态流星。
- 将视觉运动方向统一修正为左上指向右下。
- 如果固定流星参与视差，继续保留原视差关系，只修改朝向。
- 拖尾应位于流星头部的左上侧，避免出现“头尾反向”。

#### 验收

- 星空区不再出现视觉上朝右上的固定流星。
- 修正后不破坏原有背景构图和密度。

---

### 3.2 随机高速流星系统

#### 目标

新增偶发、短促、速度很快的真实流星效果。它属于基于时间的瞬时特效，不属于滚动场景内容。

#### 行为要求

- 星空区可见时才允许生成；离开星空区后停止生成并清理残留实例。
- 任意时刻最多出现 1~2 颗。
- 生成间隔随机，避免固定节奏；正式参数区间为 1~3 秒，可配置。该范围由项目所有者实际试用后确认。
- 单颗流星持续约 300~900 毫秒，速度明显快于所有背景视差元素。
- 总体方向为左上到右下，允许小范围角度随机。
- 具有亮度较高的流星头和较短拖尾。
- 尾迹快速衰减，不形成长时间停留的线条。

#### 与滚动解耦

动态流星必须满足：

- 不读取 scroll progress 作为位置计算依据；
- 不挂载在会被章节 transform 的场景容器内部；
- 不继承背景星点的整体位移；
- 页面停止滚动时仍可按时间随机出现；
- 页面快速上下滚动时，流星轨迹仍基于 viewport 连续运动。

推荐放置在星空章节上方的独立 overlay，使用局部 fixed 或不受场景 transform 影响的 absolute 容器。overlay 必须限制在星空可视区域，不得在陆地、海洋或其他章节漏出。

#### 推荐实现

优先使用独立 Canvas + `requestAnimationFrame`：

- 维护少量 meteor 实例；
- 每个实例包含起点、速度、寿命、拖尾长度和透明度；
- 通过 section 可见性控制启动与暂停；
- 页面隐藏时暂停 RAF；
- 组件卸载时清除 RAF、timer 和 observer；
- 避免热更新或重复进入章节后叠加多个循环。

DOM + CSS Animation 可以作为备选，但同样必须满足可见性控制、实例上限和资源清理。

#### 降级策略

- `prefers-reduced-motion: reduce`：关闭随机流星，保留静态背景装饰。
- 移动端或低性能设备：最大实例数降为 1，缩短拖尾或降低绘制分辨率。

#### 验收

- 流星偶发而非持续出现。
- 运动明显快速、短促、朝右下。
- 流星不会跟随页面进度上下拖动。
- 离开星空区后不再出现。
- 多次进入星空区后不会出现生成频率翻倍或内存泄漏。

### 3.3 星体与行星光环层次

#### 现状

现有光环更像两个尺寸接近的透明图层叠加，缺少清晰的明度、尺寸和前后层次。

#### 目标结构

建议将主要星体或右上行星的视觉拆为：

1. 核心主体：边缘最清楚、颜色最实；
2. 内侧亮层：紧贴主体，尺寸略大，亮度较高；
3. 外侧扩散层：尺寸明显更大，透明度明显更低；
4. 行星环或轮廓环：与主体建立明确遮挡关系；
5. 可选环境散射：非常轻，不得出现大面积模糊雾团。

#### 关键要求

- 各层尺寸差和透明度差必须明确。
- 不得仅复制同一张图后改变透明度。
- 保留像素化边缘；避免依赖大半径 CSS blur。
- 行星环需要有前后遮挡：后半环在行星后方，前半环覆盖行星前方。
- 光环不得覆盖导航或主体文案。

#### 验收

- 用户能明确感知“发光层次”，而不是两个圆重叠。
- 右上行星的主体、光晕和行星环关系清楚。
- 缩放和不同分辨率下不出现明显锯齿失控或模糊发虚。

## 4. 三层浪花与气泡优化

### 4.1 阶段归属说明

浪花位于更早的陆地到海洋过渡，但本次修改发生在 M5 完成之后，因此统一归入 M5.5，作为跨章节视觉回归与抛光任务，不再标记为 M4.5。

### 4.2 参考素材

建议将本次参考图放入：

- `wave-medium-02.png`

参考图用于确定轮廓、密度与像素语言，不要求未经处理直接作为生产资源。

### 4.3 方案结论

采用“处理后的贴图/sprite + 轻量代码控制”的混合方案。

不推荐完全算法化重绘，原因是卷浪轮廓和泡沫分布高度依赖美术形态。代码应负责：

- 三层定位与裁切；
- 不同速度的轻微横向运动；
- 小幅上下起伏；
- 平铺、错位和透明度控制；
- 响应式缩放与性能降级。

### 4.4 三层分工

#### 前景层

- 轮廓最大、最清晰、卷曲最强；
- 泡沫细节最丰富；
- 承担主要遮挡和过渡作用；
- 不得遮挡关键交互元素。

#### 中景层

- 高度与卷曲程度低于前景；
- 细节适中，用于连接前后层；
- 与前景保持不同相位和速度，避免复制感。

#### 背景层

- 最低、最轻、最简洁；
- 以海面起伏和稀疏泡沫为主；
- 透明度最低，不能抢夺主体视觉。

### 4.5 气泡与泡沫使用

- 横向气泡带用于海面边缘、浪尖泡沫和浪间碎泡。
- 竖向气泡簇用于海水内部的上升气泡，不直接机械铺在海平面。
- 气泡密度应从海面附近或海中局部出现，避免均匀分布。
- 浪花泡沫与海底上升气泡应在视觉语言上统一，但在空间用途上区分。

### 4.6 素材处理

- 生产素材需透明背景。
- 去除无关黑底、杂色和多余空白。
- 保留清晰的大像素块，不做平滑插值。
- 如需缩放，使用整数倍或接近整数倍的像素比例。
- 横向平铺前需处理接缝；可使用分段错位或多个变体降低重复感。
- 参考图中的颜色可根据现有海洋配色做统一，但不得改变整体像素语言。

### 4.7 滚动兼容

- 保留 M5 已有滚动区间和章节切换逻辑。
- 三层浪花只能在原有过渡区域内工作。
- 不得因素材高度变化导致章节跳动、进度错位或布局抖动。
- 对不同视口测试裁切，避免浪花在超宽屏或移动端露出空白接缝。

#### 验收

- 三层有明确差异，不像同一资源重复三次。
- 过渡比 M5 基线更自然、美观。
- 气泡与浪花结合合理，不显杂乱。
- 不改变 M5 原有章节节奏和交互结果。

## 5. 工程与质量要求

### 5.1 建议模块

- `MeteorOverlay`：随机动态流星；
- `StaticMeteorDecor`：固定背景流星；
- `PlanetHalo`：星体/行星多层光环；
- `WaveLayerFront` / `WaveLayerMid` / `WaveLayerBack`；
- `FoamBubbleDecor`：海面泡沫和水下气泡。

命名可适配现有项目，不强制新建所有组件；重点是职责分离。

### 5.2 参数配置

建议集中配置：

- 流星生成最小/最大间隔；
- 流星速度、角度、寿命、拖尾长度；
- 最大流星数量；
- 三层浪花速度、相位、振幅、透明度；
- 光环各层尺寸、透明度和明度。

### 5.3 回归检查

M5.5 完成后必须确认：

- 首页导航、按钮和链接可用；
- 滚动进度和章节定位与 M5 基线一致；
- 星体、流星不遮挡主体文字；
- 浪花替换未引发布局跳动；
- 桌面端、移动端和 reduced-motion 状态正常；
- 控制台无新增错误或重复动画警告。
