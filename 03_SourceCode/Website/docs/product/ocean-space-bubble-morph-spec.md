# 海洋到星空：气泡变繁星与流星规格

状态：已实施并通过正式验收
建议里程碑：M5
前置条件：M4 与 M4.5 已完成
目标：用海底浓密气泡的连续形态转换，完成深海到星空的自然过渡

当前实现参数快照（2026-07-18）：

- 时间线：`0.640–0.660` 预热、`0.660–0.690` 增密、`0.690–0.720` 增亮、`0.720–0.755` 气泡转星点、`0.755–0.785` 星点转流星、`0.785–0.800` 稳定星空；
- 统一池：固定种子 `0x50495845`，最多 180 项 `MorphParticle`，在模块初始化时一次创建；
- 桌面：三股上升流、基础密度 `0.49`、最大流星比例 `0.055`、尾迹上限 `58px`；
- 375px：两股上升流、粒子密度倍率 `0.86`、最大流星比例 `0.022`、尾迹上限 `32px`；
- Reduced Motion：取消定向流动画、时间抖动与流星，保留按进度发生的气泡增亮和静态星点转换；
- 固定方向：`SceneController.world.rotation` 始终为 `0`，所有方向变化仅修改局部坐标、形态、颜色和透明度。

---

## 1. 核心创意

海洋到星空的主要视觉机制改为：

```text
海底水流增强
→ 气泡数量逐渐增多
→ 气泡形成浓密上升流
→ 背景逐渐变暗
→ 气泡轮廓逐渐收缩并发亮
→ 同一批气泡变成星点
→ 部分高速气泡被拉伸为流星
→ 水下光束扩散成银河或星云
→ 星空稳定
```

关键要求：

- 气泡和星星必须具有视觉连续性；
- 不允许销毁全部气泡后重新生成无关星星；
- 流星必须由部分已有气泡转换；
- 用户向上滚动时能够反向变回气泡。

## 2. 与现有 M5 的关系

如果当前已经存在或历史方案曾提出：

- 三层海浪；
- 世界旋转或翻转；
- 泡沫转星点；
- 星空切换；

本轮先审查其影响，但世界旋转、翻转或倾斜不再存在保留选项。

新的主次关系：

```text
主要过渡：气泡 → 星点 → 流星
辅助过渡：颜色、光束、透明度或波纹
```

旧的 150～180 度和 15～35 度旋转方案均已失效。若代码或文档中仍有残留，必须移除，不得弱化后保留。

不得在同一短区间同时堆叠：

- 巨浪；
- 任何角度的世界旋转、翻转或倾斜；
- 高密度气泡；
- 大量流星；
- 强烈闪光。

视觉焦点必须明确。

## 4. 统一粒子对象

气泡、星点与流星应共享同一粒子对象。

推荐结构：

```ts
type MorphParticle = {
  id: number;
  seed: number;

  oceanX: number;
  oceanY: number;
  oceanRadius: number;
  oceanSpeed: number;
  oceanDrift: number;

  starX: number;
  starY: number;
  starRadius: number;
  starBrightness: number;

  meteorDirection: number;
  meteorLength: number;
  meteorEligible: boolean;

  phase: number;
  depthLayer: number;
};
```

同一对象拥有：

```text
BubbleState
StarState
MeteorState
```

通过 `morphProgress` 插值，而不是清空气泡数组再创建星星数组。

禁止：

- 回滚时重新随机；
- 每帧创建大量新粒子；
- 气泡与星星使用完全无关的身份；
- 星点瞬移到不相关位置。

## 5. 气泡浓密化

### 起始

在 Programs 主要展示区，气泡维持正常密度，不影响阅读。

### 预热

接近 M5 时：

- 海草摆动增强；
- 鱼群逐渐离开；
- 气泡速度略微增加；
- 细小气泡从海床、岩石缝隙或遗迹附近出现；
- 气泡方向逐渐一致。

### 浓密气泡流

在 `0.660–0.690`：

- 气泡数量明显增加；
- 形成一到三股上升流；
- 中小气泡构成主要密度；
- 近景大气泡数量受控；
- Programs 内容逐渐退出主要视野。

“浓密”不等于随机铺满屏幕，应形成有方向、有聚集区域的气泡流。

## 6. 气泡变星点

每个气泡逐步发生：

```text
半透明圆环
→ 圆环变细
→ 内部高光增强
→ 外圈消失
→ 中心缩小
→ 成为发光像素星点
```

可插值：

- 半径；
- 透明度；
- 描边厚度；
- 中心亮度；
- 颜色；
- 上升速度；
- 轻微位置偏移。

颜色建议：

```text
海蓝/青色
→ 浅青白
→ 暖白或冷白星光
```

避免所有粒子在同一时刻纯白闪烁。

星点最终位置可与气泡轨迹略有偏移，但必须保持连续，不能瞬移。

---

## 7. 部分星点变流星

只有少量粒子设置：

```ts
meteorEligible: true
```

建议比例：

```text
桌面端：3%～8%
移动端：1%～3%
Reduced Motion：0%
```

选择必须由固定种子决定。

形成过程：

```text
明亮星点
→ 速度增加
→ 沿固定像素轨迹加速
→ 像素尾迹拉长
→ 成为流星
```

要求：

- 不突然从屏幕外大量飞入；
- 尾迹像素化；
- 尾迹长度有限；
- 不使用强烈模糊；
- 不遮挡关于我和导航；
- 方向总体一致，只允许轻微差异。

---

## 8. 水下环境退出

随着气泡转星：

- 鱼群逐渐消失；
- 海草和珊瑚降低对比度；
- 海床向下远离；
- 水体悬浮颗粒变为星尘；
- 深海蓝逐渐转为靛蓝和紫色；
- 顶部光束扩散为银河或星云。

不得：

- 海洋元素同一帧消失；
- 背景直接切黑；
- 气泡仍处于水下状态时突然出现完整星空内容。

---

## 9. 光束变银河

水下光束作为第二条连续线索：

```text
水下光束
→ 逐渐变宽
→ 边缘出现星尘
→ 颜色转为蓝紫
→ 成为银河带或星云
```

要求：

- 保持像素风；
- 光束与星云方向连续；
- 不出现突兀的大块贴图；
- 不要求写实银河，但要有形态继承。

---

## 10. 背景颜色

建议：

```text
深海藏青
→ 接近黑色的深蓝
→ 靛蓝
→ 紫蓝星空
```

推荐变量：

```css
--morph-deep-sea: #04162f;
--morph-night-blue: #07112d;
--morph-indigo: #181443;
--morph-purple: #32245d;
--morph-nebula: #5b4cb5;
--morph-star: #fff3c4;
```

禁止：

- 瞬间切黑；
- 全屏白闪；
- 亮青色一直保留到星空阶段。

---

## 11. 固定世界方向

背景世界旋转或翻转已作为 M4.5 需求更正全面取消，不再存在“弱化保留”方案。

以下约束适用于标准动效、移动端和 Reduced Motion：

- Canvas 世界方向始终固定为 0°；
- 导航、正文和固定 UI 始终保持水平；
- 向下推进和向上倒放均不得出现旋转、翻转或倾斜；
- 水流、光束、银河和流星只能通过坐标、形态、颜色与透明度变化表达方向；
- 不得通过旋转世界容器、场景容器或粒子容器制造转场。

流星可以沿固定像素轨迹移动，但这只是粒子坐标变化，不得改变世界或容器方向。

## 12. DOM 与固定 UI

以下始终保持水平：

- 顶部导航；
- 右侧阶段进度；
- Programs 按钮；
- 关于我内容；
- 可访问文本。

要求：

- Programs 内容自然退出；
- 关于我内容在星空稳定后进入；
- 粒子层不能阻止点击；
- 粒子不能长时间遮挡正文；
- 不允许焦点丢失。

---

## 13. 可逆性

向上滚动时必须完整倒放：

```text
星空
→ 流星尾迹缩短
→ 流星变星点
→ 星点重新出现气泡外圈
→ 星点变气泡
→ 气泡恢复透明水体质感
→ 海床、海草和鱼群恢复
→ 深海稳定
```

不得：

- 星星瞬间消失；
- 流星突然停止；
- 气泡重新随机；
- 星云瞬间变光束；
- 背景闪色；
- 回滚后的粒子数量变化。

---

## 14. 移动端

移动端：

- 降低粒子数量；
- 减少浓密气泡流数量；
- 流星比例降低；
- 尾迹更短；
- 不使用任何角度的世界旋转、翻转或倾斜；
- 保留气泡变星点这一核心过程；
- 不允许降级成直接切换背景。

---

## 15. Reduced Motion

Reduced Motion 下：

```text
深海静态场景
→ 气泡亮度逐渐增强
→ 少量气泡变静态星点
→ 星空静态场景
```

取消：

- 流星；
- 大量高速粒子；
- 世界旋转；
- 强烈闪光；
- 高速视差。

---

## 16. 性能

- 使用统一粒子对象池；
- 使用固定随机种子；
- 流星尾迹复用；
- 不每帧创建新数组；
- 分别配置桌面与移动端数量；
- 页面隐藏时暂停；
- DPR 上限保持 2；
- 粒子密度高峰设置上限；
- 不使用全屏视频；
- 不引入新的大型依赖。
