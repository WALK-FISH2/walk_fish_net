# 陆海与海空过渡精修需求文档

状态：M4.5、M5 均已实施并通过正式验收
适用项目：Pixel//Walk 个人像素叙事网站
建议文件路径：`docs/product/transition-polish-spec.md`
对应里程碑：M4.5、M5
前置条件：M1、M2、M3、M4 已完成

---

## 1. 文档目的

本文件统一定义首页两段核心过渡的精修要求：

1. **陆地 → 海洋**：使用翻涌海浪、泡沫和镜头穿浪遮挡场景接缝，让进入水下更加自然；
2. **深海 → 星空**：使用海底逐渐变浓密的气泡，连续变形成繁星与流星，让海洋与星空形成视觉继承关系。

本文件是这两段过渡的主要产品与验收依据。后续 Codex 提示词只需引用本文件，不应在聊天中重复全部需求。

---

## 2. 总体原则

### 2.1 不推翻现有成果

当前陆地、下潜、深海和星空主体场景已经存在。

本轮只精修两段过渡：

```text
M4.5：陆地 → 海洋
M5：深海 → 星空
```

不得：

- 重建项目；
- 更换 Astro 架构；
- 修改文章内容系统；
- 修改 Programs 内容模型；
- 修改 `/programs` 主路由；
- 修改旧 `/projects` 兼容逻辑；
- 引入传统后端；
- 修改 `constitution.md`；
- 使用全屏视频替代场景；
- 使用受版权保护的游戏或电影素材。

### 2.2 单一滚动进度

所有关键动画继续由统一的全局滚动进度控制：

```ts
type StoryProgress = number; // 0..1
```

要求：

- 向下滚动推进；
- 向上滚动完整倒放；
- 用户停止滚动时，叙事状态停留在当前进度；
- 不劫持滚轮；
- 不强制吸附；
- 不用一次性播放动画代替滚动映射；
- 同一进度应得到基本一致的画面。

### 2.3 Canvas 与 DOM 分离

Canvas / PixiJS 负责：

- 海浪；
- 泡沫；
- 水雾；
- 水面；
- 气泡；
- 星点；
- 流星；
- 银河；
- 星云；
- 环境粒子；
- 场景色彩与视差。

DOM 负责：

- 导航；
- Programs 内容；
- 文章内容；
- 按钮；
- 标签；
- 关于我；
- 无障碍文本；
- 错误提示。

Canvas 世界、固定 UI 和正文在所有进度与动效模式下都必须保持水平。海洋到星空只允许通过颜色、光束、透明度和粒子形态平滑过渡，不得旋转、翻转或倾斜背景世界。

# 第一部分：M4.5 陆地到海洋翻涌海浪

## 3. 目标效果

当前陆地到深海已经具有水位上涨和下潜结构。

本轮将它升级为：

```text
陆地环境逐渐变冷
→ 远处水面出现
→ 第一层小浪出现
→ 三层浪潮逐渐接近
→ 前浪与泡沫翻涌
→ 泡沫和水雾遮挡场景接缝
→ 镜头穿过浪峰
→ 水下色调接管画面
→ 水面退到屏幕顶部
→ 深海稳定
```

翻涌海浪的主要作用不是单纯增加装饰，而是：

- 给水位上涨增加视觉原因；
- 遮挡陆地和水下素材切换；
- 让镜头进入水下更自然；
- 避免直接切换背景颜色；
- 保持陆地与深海之间的空间连续性。

---

## 4. M4.5 进度区间

保持现有 M4 全局边界不变，建议内部节奏：

```text
0.300–0.315  陆地环境变冷
0.315–0.328  远处水面和第一层小浪出现
0.328–0.342  浪潮接近，水位继续上涨
0.342–0.355  三层浪峰翻涌并经过屏幕中部
0.355–0.368  泡沫、水雾覆盖接缝，镜头穿浪
0.368–0.380  完全入水，水面退到屏幕顶部
```

若当前代码已经采用略有不同的区间，可以在不改变 M4 与 M5 全局边界的前提下微调。

所有区间必须集中在现有故事配置中，不得散落为多个魔法数字。

建议配置结构：

```ts
export const LAND_OCEAN_TRANSITION = {
  cooling: [0.300, 0.315],
  farWave: [0.315, 0.328],
  waveApproach: [0.328, 0.342],
  waveBreak: [0.342, 0.355],
  foamCover: [0.355, 0.368],
  underwaterSettle: [0.368, 0.380],
} as const;
```

---

## 5. 三层海浪

至少实现远浪、中浪和前浪。

### 5.1 远浪

作用：

- 最先出现；
- 告诉访问者水域正在接近；
- 建立水面的空间方向。

要求：

- 振幅较小；
- 颜色较暗；
- 速度较慢；
- 泡沫很少；
- 位于场景后方；
- 对比度低于中浪和前浪。

### 5.2 中浪

作用：

- 形成主要水面轮廓；
- 承担主要水位上涨视觉；
- 连接远浪和前浪。

要求：

- 振幅中等；
- 波峰轮廓清晰；
- 有少量高光；
- 具有轻微泡沫；
- 速度高于远浪；
- 是主要水面层。

### 5.3 前浪

作用：

- 形成过渡高潮；
- 覆盖场景接缝；
- 帮助镜头穿浪。

要求：

- 最后进入；
- 振幅最大；
- 速度最快；
- 具有明显泡沫；
- 可以短暂遮挡画面中下部；
- 不能持续遮挡导航或正文；
- 入水完成后自然退出或退到顶部。

### 5.4 三层差异

三层浪不能只是同一条正弦曲线复制三次。

至少要在以下参数上存在明显差异：

```text
振幅
周期
相位
水平速度
透明度
颜色
像素块尺寸
泡沫密度
波峰数量
```

---

## 6. 像素浪峰

目标是“像素化翻涌浪峰”，不是平滑矢量波浪。

要求：

- 波峰边缘呈阶梯状；
- 波谷具有轻微不规则变化；
- 局部可出现卷曲浪头；
- 波浪不能只是一条水平线；
- 不要求真实流体模拟；
- 保持原创；
- 不使用第三方海浪视频；
- 不直接复制经典游戏素材。

可以使用：

- PixiJS Graphics；
- 程序化折线；
- 曲线采样后像素化；
- 原创像素纹理平铺；
- 固定种子的轻微噪声。

---

## 7. 泡沫与水雾

### 7.1 泡沫位置

泡沫主要出现在：

- 波峰边缘；
- 前浪破碎位置；
- 镜头穿浪时；
- 场景切换接缝附近；
- 屏幕中部的过渡遮挡区域。

### 7.2 泡沫形态

要求：

- 使用小型像素簇；
- 大小有变化；
- 不全部使用圆形；
- 颜色由浅青到白；
- 使用固定随机种子；
- 可逆；
- 使用对象池或复用机制；
- 不每帧大量创建和销毁。

建议泡沫对象：

```ts
type FoamParticle = {
  id: number;
  seed: number;
  x: number;
  y: number;
  size: number;
  drift: number;
  opacity: number;
  layer: number;
  phase: number;
};
```

### 7.3 水雾

水雾只作为辅助：

- 透明度低；
- 生命周期短；
- 不模糊正文；
- 不持续覆盖导航；
- 不阻止点击；
- 移动端减少或取消；
- Reduced Motion 下取消。

---

## 8. 场景遮挡与镜头穿浪

当泡沫覆盖画面中部时：

1. 陆地背景逐渐降低对比度；
2. 草地、树木和山丘逐渐变成偏蓝剪影；
3. 水下色调由下向上接管；
4. 水下光束和气泡开始出现；
5. 泡沫覆盖关键接缝；
6. 泡沫退去后，画面已经位于水下；
7. 水面继续向顶部退去；
8. 深海场景稳定。

镜头穿浪可以通过以下组合实现：

- 前浪向上推进；
- 场景世界轻微纵向位移；
- 折射只作用于水线附近；
- 泡沫覆盖接缝；
- 水下颜色从下向上扩展；
- 顶部水面逐渐远离。

禁止：

- 白屏；
- 瞬间换背景；
- 任何世界旋转；
- 持续镜头抖动；
- 强制滚动吸附；
- 自动播放完整动画；
- 导航或正文参与旋转；
- DOM 长时间不可读。

---

## 9. M4.5 可逆性

向上滚动时必须完整倒放：

```text
深海
→ 水面从顶部重新靠近
→ 泡沫重新汇聚
→ 前浪退回
→ 中浪退回
→ 远浪退回
→ 陆地颜色与细节恢复
```

不得：

- 重新随机浪花；
- 泡沫突然消失；
- 波浪相位跳变；
- 陆地最后一帧突然出现；
- 反向过程缺少某一层浪；
- 回滚后粒子数量发生变化。

---

## 10. M4.5 移动端

移动端必须保留：

- 远浪；
- 主浪；
- 少量泡沫；
- 镜头穿浪；
- 水面退到顶部。

可以简化：

- 前浪细节；
- 泡沫密度；
- 水雾；
- 折射；
- 视差；
- 粒子数量。

不得简化为直接切色。

---

## 11. M4.5 Reduced Motion

Reduced Motion 下使用：

```text
陆地静态场景
→ 单层像素浪幕向上遮挡
→ 水下静态场景
```

取消：

- 多层高速浪；
- 水雾；
- 大幅视差；
- 动态折射；
- 镜头晃动。

仍必须保留明确的陆海交接感。

---

## 12. M4.5 性能

- 波浪几何复用；
- 泡沫对象池复用；
- 不每帧创建大量对象；
- DPR 上限保持 2；
- 页面隐藏时暂停环境循环；
- 移动端减少泡沫；
- 不增加全屏视频；
- 不引入新的大型动画框架；
- 场景销毁时清理资源和监听器。

# 第二部分：M5 深海气泡变繁星与流星

## 15. 核心创意

海洋到星空的主要视觉机制调整为：

```text
深海水流增强
→ 气泡数量逐渐增加
→ 气泡形成浓密上升流
→ 背景逐渐变暗
→ 气泡轮廓收缩并发亮
→ 同一批气泡变成繁星
→ 少量繁星拉伸为流星
→ 水下光束扩散为银河或星云
→ 星空稳定
```

核心要求：

- 气泡、星点和流星之间有连续的形态继承；
- 不允许清空气泡后重新生成一批无关星星；
- 流星由部分已有粒子转换；
- 向上滚动时，流星和星星能够变回气泡；
- 过渡主角是“气泡变繁星和流星”。

---

## 16. M5 与固定世界方向的关系

需求更正：背景世界旋转或翻转已全面取消。该更正归入 M4.5，M5 不得重新引入任何角度的世界旋转。

海洋到星空的主次关系：

```text
主要过渡：气泡 → 星点 → 流星
辅助过渡：颜色、光束、透明度和环境元素退出
```

场景方向必须满足：

- Canvas 世界方向固定为 0°；
- DOM、导航和正文保持水平；
- 桌面、移动端、标准动效和 Reduced Motion 使用相同的固定方向约束；
- 向下推进和向上倒放均不得出现旋转；
- 旧的 150～180 度和 15～35 度旋转方案均不再有效。

不得在同一短区间同时堆叠：

- 巨浪；
- 世界旋转；
- 高密度气泡；
- 大量流星；
- 强烈闪光。

所有动效模式下都必须完全取消世界旋转。

---

## 17. M5 进度区间

建议：

```text
0.640–0.660  深海预热，气泡数量开始增加
0.660–0.690  浓密气泡流形成
0.690–0.720  背景变暗，气泡发亮
0.720–0.755  气泡轮廓收缩并转为星点
0.755–0.785  部分星点拉伸为流星
0.785–0.820  银河与星云建立，星空稳定
```

若现有 M5 固定为 `0.66–0.80`，可以在不破坏前后阶段的前提下压缩最后稳定段。

建议配置：

```ts
export const OCEAN_SPACE_MORPH = {
  preheat: [0.640, 0.660],
  bubbleDensity: [0.660, 0.690],
  brighten: [0.690, 0.720],
  bubbleToStar: [0.720, 0.755],
  starToMeteor: [0.755, 0.785],
  settleSpace: [0.785, 0.820],
} as const;
```

## 18. 统一 MorphParticle

气泡、星点和流星应共享同一粒子对象。

建议：

```ts
type MorphParticle = {
  id: number;
  seed: number;

  oceanX: number;
  oceanY: number;
  oceanRadius: number;
  oceanSpeed: number;
  oceanDrift: number;

  starX: number;
  starY: number;
  starRadius: number;
  starBrightness: number;

  meteorDirection: number;
  meteorLength: number;
  meteorEligible: boolean;

  phase: number;
  depthLayer: number;
};
```

同一对象具有：

```text
BubbleState
StarState
MeteorState
```

通过局部进度插值，不允许：

- 清空气泡数组；
- 新建完全无关的星星数组；
- 回滚时重新随机；
- 每帧创建大量粒子对象；
- 星点瞬移到无关位置。

---

## 19. 气泡逐渐浓密

### 19.1 深海稳定区

在 Programs 主要展示区域，气泡保持正常密度，不影响阅读。

### 19.2 预热区

接近 M5 时：

- 海草摆动增强；
- 鱼群逐渐离开；
- 气泡速度略微增加；
- 细小气泡从海床、岩石和遗迹附近出现；
- 气泡方向逐渐趋于一致；
- Programs 内容逐渐退出主要视野。

### 19.3 浓密上升流

在 `0.660–0.690`：

- 气泡数量明显增加；
- 形成一到三股上升流；
- 中小气泡是主要密度来源；
- 近景大气泡数量受控；
- 气泡分布有方向和聚集区域；
- 不允许随机铺满整个屏幕。

---

## 20. 气泡变繁星

每个气泡逐步发生：

```text
半透明圆环
→ 圆环变细
→ 内部高光增强
→ 外圈消失
→ 中心缩小
→ 成为发光像素星点
```

可插值：

- 半径；
- 透明度；
- 描边厚度；
- 中心亮度；
- 颜色；
- 上升速度；
- 轻微位置。

颜色变化：

```text
海蓝/青色
→ 浅青白
→ 暖白或冷白星光
```

不得让所有气泡在同一时刻纯白闪烁。

星点目标位置可以略微偏离气泡轨迹，但必须：

- 保持整体运动方向；
- 不瞬移；
- 不随机跳到无关位置；
- 向上滚动时沿原路径返回。

---

## 21. 繁星变流星

只有少量粒子设置：

```ts
meteorEligible: true
```

建议比例：

```text
桌面端：3%～8%
移动端：1%～3%
Reduced Motion：0%
```

具体比例应根据实际帧率调整，选择必须由固定种子决定。

形成过程：

```text
明亮星点
→ 速度增加
→ 沿固定像素轨迹加速（不旋转世界或粒子容器）
→ 像素尾迹拉长
→ 成为流星
```

要求：

- 流星由已有星点转换；
- 不突然从屏幕外大量飞入；
- 尾迹像素化；
- 尾迹长度有限；
- 不使用强烈模糊；
- 不遮挡关于我和导航；
- 运动方向总体一致，仅有轻微差异。

---

## 22. 深海环境退出

随着气泡变星：

- 鱼群逐渐消失；
- 海草与珊瑚降低对比度；
- 海床向下远离；
- 水体悬浮颗粒变为星尘；
- 深海蓝转为接近黑色的深蓝；
- 再逐渐转为靛蓝与紫色；
- 水下光束扩散为银河或星云。

不得：

- 所有海洋元素同一帧消失；
- 背景直接切黑；
- 气泡仍处于水下状态时突然显示完整星空内容；
- 使用全屏白闪遮盖过渡。

## 23. 水下光束变银河或星云

水下光束作为第二条连续线索：

```text
水下光束
→ 逐渐变宽
→ 边缘出现星尘
→ 颜色转为蓝紫
→ 成为银河带或星云
```

要求：

- 保持像素风；
- 光束与银河方向连续；
- 不出现突兀的大块贴图；
- 不要求写实银河；
- 但必须看出形态继承。

推荐色彩：

```css
--morph-deep-sea: #04162f;
--morph-night-blue: #07112d;
--morph-indigo: #181443;
--morph-purple: #32245d;
--morph-nebula: #5b4cb5;
--morph-star: #fff3c4;
```

## 25. M5 可逆性

向上滚动时：

```text
星空
→ 流星尾迹缩短
→ 流星变星点
→ 星点重新出现气泡外圈
→ 星点变气泡
→ 气泡恢复水体透明质感
→ 海床、海草和鱼群恢复
→ 深海稳定
```

不得：

- 星星瞬间消失；
- 流星突然停止；
- 气泡重新随机；
- 星云瞬间变光束；
- 背景闪色；
- 回滚后粒子数量变化；
- 回滚后气泡位置完全不同。

---

## 26. M5 移动端

移动端：

- 减少粒子数量；
- 减少气泡流数量；
- 流星比例降低；
- 尾迹更短；
- 不使用任何世界旋转；
- 保留气泡变星点这一核心过程；
- 不允许降级为直接切换背景。

---

## 27. M5 Reduced Motion

Reduced Motion 下：

```text
深海静态场景
→ 气泡亮度逐渐增强
→ 少量气泡变为静态星点
→ 星空静态场景
```

取消：

- 流星；
- 高密度高速粒子；
- 世界旋转（此约束同样适用于标准动效和移动端）；
- 强烈闪光；
- 高速视差。

- 页面隐藏时暂停；
- DPR 上限保持 2；

- 不使用全屏视频；
- 不引入新的大型依赖；
