# 滚动交互规格 Interaction Spec

## 1. 全局进度

```text
globalProgress ∈ [0, 1]
```

```text
0.00–0.30 陆地
0.30–0.38 下潜
0.38–0.66 深海
0.66–0.80 海洋到星空
0.80–1.00 星空
```

## 2. 确定性

同一滚动位置应得到基本一致的场景颜色、元素位置、粒子转换状态、DOM 阶段和导航状态。Canvas 世界方向固定为 0°；环境循环可以随时间变化，但不得改变叙事状态。

## 3. 正向和反向

```text
向下：陆地 → 深海 → 星空
向上：星空 → 深海 → 陆地
```

不得在回滚时瞬移、重新生成全部粒子、闪白或让导航错位。

## 4. 视差

```text
远景 0.10–0.20
中景 0.30–0.50
近景 0.60–0.80
前景 0.90–1.20
```

陆地引导人物沿地面移动时，DOM 首屏面板和文章路牌都必须在人物路线之上保留连续可见走廊。允许装饰性 Canvas 元素短暂经过 DOM 背后，但引导人物不得在陆地阶段的大部分行程中被 Hero 或三张文章路牌遮住。窄屏无法同时横排三张路牌时，使用可横向滚动、可聚焦的路牌轨道，不把三张卡片纵向压入地面走廊。

当前 Programs 主阅读区采用顺序文档流，这是已经验证的防重叠基线。M6.1 只在有足够内容宽度的桌面完整动画中把该区域升级为两条隔离轨道：标题在左上安全区进行 section 内受限停驻，三张档案终端整体位于右侧，但继续使用当前完整纵向 Grid 和普通文档流。不得给单张卡片增加 sticky、独立滚动步骤或接棒式显隐。移动端、宽度不足的视口和简化动画继续使用标题在上、三张卡片在下的顺序文档流。

## 5. 陆地下潜

| 全局进度 | 事件 |
|---:|---|
| 0.300 | 陆地稳定，下潜段开始 |
| 0.300–0.315 | 环境变冷、变暗，陆地降低对比 |
| 0.315–0.328 | 远浪从视口底部外先出现，水面逐渐进入视口 |
| 0.328–0.342 | 中浪接近，水线连续上升至屏幕中部 |
| 0.342–0.355 | 前浪翻涌并经过内容区，三层像素浪明确分离，DOM 仅产生轻量折射 |
| 0.355–0.368 | 泡沫遮挡水陆接缝，前浪越过镜头，陆地转为低对比剪影后淡出 |
| 0.368–0.380 | 完全入水，镜头继续下潜，水面保留在顶部 |
| 0.380–0.430 | 浅层海水，顶部水面逐渐退出，光束和气泡建立深度 |
| 0.430–0.500 | 深海环境稳定并进入 Programs 档案区域 |
| 0.500–0.590 | Programs 主要阅读区 |
| 0.590–0.640 | 深海尾段 |
| 0.640–0.660 | M4 预热：水流、海草、气泡轻量增强，鱼群离开，光束形态轻微偏移但不旋转场景容器 |

水面位置、可见度、折射强度、陆地退场和镜头下潜必须由同一个 `globalProgress` 计算。Canvas 水面与 DOM 折射带读取同一套纯函数，向上滚动时不得创建新的水线、泡沫、鱼群或气泡状态。

M4.5 的远浪、中浪和前浪必须保持独立层与独立参数。M5.5 以处理后的透明 sprite 作为三层主轮廓，TilingSprite 只做平铺、相位、轻微水平漂移和上下起伏；原 Graphics 轮廓只作为素材解码失败降级。横向泡沫带贴合浪尖与接缝，竖向气泡簇只进入水体内部，既有固定泡沫池缩减为局部补充。移动端降低 sprite 高度、泡沫和气泡数量，Reduced Motion 只保留单层静态像素浪幕。

深海层级统一按远景、中景、近景、前景配置；Programs 档案由真实 DOM 承载，必须显示 Program 状态、演示类型、技术栈、当前限制和真实链接，并保持“本人编写的程序”语义。

### 5.1 M6.1 Programs 左右分栏与标题渐隐

M6.1 不改变上述全局进度区间。桌面完整动画在 `0.430–0.640` 内完成以下阅读节奏：

| 全局进度 | 内容层目标 |
|---:|---|
| 0.430–0.460 | 标题进入 Programs section 左上安全锚点，完整卡片流进入右侧轨道 |
| 0.460–0.590 | 标题受 section 边界约束停驻，三张卡片保持当前完整纵向排列并自然滚动 |
| 0.590–0.610 | 最后一张卡片进入尾段，标题从深海亮色逐渐变暗并降低透明度 |
| 0.610–0.640 | 最后一张卡片和总入口自然划过；标题完全隐去，为 M5 边界让位 |

三张卡片继续使用一个 `.portholes` 普通文档流容器，不增加单卡停驻或显示状态。标题是左侧轨道内的受限 sticky，不是跨 section 的 fixed UI；标题退场由单一、可逆的局部进度驱动，允许颜色、透明度和极轻的向上位移变化，不允许旋转或闪烁。不得新增第二个滚动状态机、页面级 pin、scroll lock 或整页吸附。

移动端竖屏、内容宽度不足和简化动画不执行上述停驻：保持标题 → 卡片 01 → 卡片 02 → 卡片 03 → 总入口的普通文档流。正向与反向滚动、模式切换和 Canvas fallback 都必须保持同一 DOM 与键盘顺序。完整参数和验收点见 `docs/product/m6-1-underwater-programs-layout-spec.md`。

### 5.2 M6.2 跨世界小章鱼状态

M6.2 不改变五段全局进度，只在现有区间中增加统一主题角色：

| 全局进度 | 角色目标 |
|---:|---|
| `0.000–0.300` | 普通小章鱼沿现有陆地路线移动，完整动画叠加轻微待机浮动 |
| `0.300–0.328` | 向上跃起，在远浪出现期间到达高点 |
| `0.328–0.355` | 向中浪和前浪落下 |
| `0.355–0.368` | 由现有泡沫/前浪遮挡，普通形态切换为潜水形态 |
| `0.368–0.380` | 潜水小章鱼建立入水状态 |
| `0.380–0.620` | 潜水小章鱼从屏幕右上逐渐移动到左下，完整动画叠加轻微浮动 |
| `0.620–0.660` | 潜水小章鱼向下潜出屏幕 |
| `0.660–0.800` | 角色缺席，保留现有 M5 粒子形变的清晰度 |
| `0.800–0.830` | 宇航员小章鱼从屏幕下方升入并恢复亮度/透明度 |
| `0.830–1.000` | 宇航员在可用视口内进行 DVD 式漂浮和边界反弹 |

普通/潜水形态的主路径、跳跃弧线、形态、亮度和可见度必须由单一 `globalProgress` 派生。待机浮动与宇航员 DVD 漂浮可以随时间变化，但不得改变叙事状态、章节进度或滚动位置。

宇航员的回滚行为分为两个明确分支：

1. 从稳定星空向上进入 `0.830–0.800` 时，宇航员继续当前边界反弹，同时逐渐变暗并降低透明度。若在完全消失前重新向下，继续同一位置、速度和循环，并平滑恢复正常颜色与透明度；不得重新入场或创建第二个 RAF。
2. 若继续向上直到宇航员完全消失，则清理或暂停该漂浮循环；进入海洋尾段时，潜水小章鱼按 `0.660–0.620` 的逆向路径从屏幕下方升起。之后重新向下进入星空时，潜水形态先下潜离场，宇航员再从屏幕下方重新升入并建立一套新的单一漂浮循环。

宇航员不得从屏幕上方下降。完全消失前的方向反转不重置坐标；完全消失后的再次进入才执行完整的下方入场。

角色视觉层只负责装饰，使用 `pointer-events: none` 和 `aria-hidden`。宇航员位于星空背景、右上行星和信号站天线/卫星装饰前方，同时位于信号站文字卡片、星座入口、导航、按钮和 fixed UI 后方。普通和潜水形态继续处于 DOM 内容层后方，并利用现有浪花/泡沫遮挡完成入水切换。

375px 缩小角色、跳跃和反弹范围，但保持相同顺序；简化动画关闭持续浮动和 DVD 反弹，只保留必要的位置、形态和淡入淡出变化。完整规格见 `docs/product/m6-2-octopus-traveler-spec.md`。

## 6. 海洋到星空

| 全局进度 | 事件 |
|---:|---|
| 0.64–0.66 | 深海尾段预热，既有固定粒子池开始增密 |
| 0.66–0.69 | 桌面三股、移动端两股定向气泡流形成 |
| 0.69–0.72 | 气泡增亮，深海颜色连续转为夜蓝 |
| 0.72–0.755 | 同一 `MorphParticle` 的气泡轮廓收缩为像素星点 |
| 0.755–0.785 | 少量既有星点沿固定像素轨迹成为流星，尾迹逐段增长 |
| 0.785–0.80 | 水下光束完成银河/星云形态，Programs 退出，About 进入并稳定 |

气泡、星点和流星共享模块级固定 `MorphParticle` 池，通过全局进度插值 ocean、star 和 meteor 状态，不销毁后重建，也不在倒放时重新随机。M4.5 的三层浪在 38% 前已经完成，M5 不重复绘制浪花。

M5.5 保留固定 `MorphParticle` 流星的数量、位置和视差关系，只把方向统一为左上到右下，拖尾位于头部左上。另有一个不进入场景树的 fixed Canvas 瞬时特效层：它只使用“当前是否处于星空”作为生命周期门，不读取 `globalProgress` 计算坐标；星空激活时以 `1–3s` 的随机间隔生成最多 1–2 颗、生命周期 `300–900ms` 的高速流星。该间隔由项目所有者于 2026-07-19 实际试用后确认；离开章节、页面隐藏、Reduced Motion 和组件卸载仍会清理 timer、RAF 与残留实例。

海洋到星空的连续性由颜色、透明度、光束和粒子形态继承完成，不允许旋转、翻转或倾斜 Canvas 世界。

## 7. DOM 稳定

使用独立容器：

```text
sceneWorld
contentOverlay
fixedUI
```

`sceneWorld`、`contentOverlay` 和 `fixedUI` 始终保持水平。三者可以独立淡入、淡出或改变局部元素位置，但不得进行世界级旋转。

## 8. 输入

支持鼠标滚轮、触控、触控板、键盘 PageDown/PageUp 和进度节点点击。

禁止禁用默认滚动、长时间阻塞主线程、强制整页吸附和无限惯性滞后。

## 9. 动效模式与 Reduced Motion

### 9.1 模式解析

首页只有一个最终动效模式状态，所有场景、DOM、动态流星和阶段高度共同读取该状态，不允许各组件分别解析媒体查询或 URL。

```text
有效 URL 参数 ?motion=full 或 ?motion=reduce
→ 用户主动保存的 full 或 reduce
→ 产品默认 full
```

- 普通网址在没有已保存选择时直接使用完整动画；
- URL 参数只覆盖当前页面，不修改本地保存值；
- 用户通过界面开关选择模式时，保存新值并退出 URL 临时覆盖状态；
- 未知的 `motion` 参数按无参数处理；
- 本地存储读取或写入失败时静默退化，不影响页面内容、滚动或 Canvas 降级。

### 9.2 模式开关

开关位于 fixed UI，不随场景滚动离开视口。可见文案使用“动画：完整”或“动画：简化”，具备可见焦点、触控尺寸和明确的无障碍名称。系统命中 `prefers-reduced-motion: reduce` 且当前为完整模式时，控件补充“系统建议简化动画”的短提示，但不弹窗、不阻塞浏览，也不自动更改首次默认模式。

切换时保持当前全局滚动进度，不跳回顶部，不修改 URL 路由、章节区间和内容顺序。现有时间线、动态流星 RAF/timer 和媒体查询监听必须先协调或清理，任意时刻只能有一套活动实例。

模式切换必须被视为一个可取消事务：记录切换前的 `globalProgress`，等待新模式布局稳定，刷新唯一的 ScrollTrigger，再依据该 ScrollTrigger 实际测得的 `start/end` 恢复等价坐标。不得使用与故事时间线范围不一致的页面总高度作为最终映射依据。若切换期间又发生新切换，旧事务必须失效，旧 RAF、refresh 或滚动恢复不得在之后覆盖新模式。

切换完成必须同时满足：

```text
abs(progressAfter - progressBefore) <= 0.01
仍可继续向下滚到 progress >= 0.999 / phase=space
仍可向上滚回 progress <= 0.001 / phase=land
页面物理底部不能对应 sea 或 transition
滚轮、触控、触控板、PageDown/PageUp 和阶段节点均未被锁定
```

### 9.3 简化动画表现

```text
陆地静态场景
→ 单层静态像素浪幕
→ 深海静态场景
→ 像素遮罩
→ 星空静态场景
```

取消快速视差、镜头摇晃和大规模粒子涌动；背景世界在标准动效与 Reduced Motion 下都固定为 0°。

已保存为简化动画时，模式必须在场景和非滚动时间动画启动前生效，不能先播放完整动画再切换。`?motion=reduce` 用于直接验收相同分支；`?motion=full` 保留用于强制验收完整分支。

### 9.4 星空内容入口

星空 DOM 中既有的前两篇文章和前两个 Program 构成四项星座入口。桌面端四项在信号站左侧/左下安全区纵向排列，每项的发光菱形星与标题都默认可见；完整动画、简化动画、系统建议简化但仍选择完整动画三种状态不得改变其内容和默认可见性。hover/focus 只提供增强反馈。375px 下四项进入信号站后的正常文档流，保持顺序、触控尺寸、键盘焦点和可读标题。
