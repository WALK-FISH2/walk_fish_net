# 滚动交互规格 Interaction Spec

## 1. 全局进度

```text
globalProgress ∈ [0, 1]
```

```text
0.00–0.30 陆地
0.30–0.38 下潜
0.38–0.66 深海
0.66–0.80 海洋到星空
0.80–1.00 星空
```

## 2. 确定性

同一滚动位置应得到基本一致的场景颜色、元素位置、旋转角度、粒子转换状态、DOM 阶段和导航状态。环境循环可以随时间变化，但不得改变叙事状态。

## 3. 正向和反向

```text
向下：陆地 → 深海 → 星空
向上：星空 → 深海 → 陆地
```

不得在回滚时瞬移、重新生成全部粒子、闪白或让导航错位。

## 4. 视差

```text
远景 0.10–0.20
中景 0.30–0.50
近景 0.60–0.80
前景 0.90–1.20
```

## 5. 陆地下潜

| 全局进度 | 事件 |
|---:|---|
| 0.300 | 陆地稳定，下潜段开始 |
| 0.300–0.315 | 环境变冷、变暗，陆地降低对比 |
| 0.315–0.330 | 水面从视口底部外出现 |
| 0.330–0.345 | 水线连续上升至屏幕中部 |
| 0.345–0.360 | 水线经过内容区，DOM 仅产生轻量像素折射 |
| 0.360–0.370 | 完全入水，陆地转为低对比剪影后淡出 |
| 0.370–0.380 | 镜头继续下潜，水面保留在顶部并开始远去 |
| 0.380–0.430 | 浅层海水，顶部水面逐渐退出，光束和气泡建立深度 |
| 0.430–0.500 | 深海环境稳定并进入 Programs 档案区域 |
| 0.500–0.590 | Programs 主要阅读区 |
| 0.590–0.640 | 深海尾段 |
| 0.640–0.660 | M4 预热：水流、海草、气泡轻量增强，鱼群离开，光束略微倾斜 |

水面位置、可见度、折射强度、陆地退场和镜头下潜必须由同一个 `globalProgress` 计算。Canvas 水面与 DOM 折射带读取同一套纯函数，向上滚动时不得创建新的水线、鱼群或气泡状态。

深海层级统一按远景、中景、近景、前景配置；Programs 档案由真实 DOM 承载，必须显示 Program 状态、演示类型、技术栈、当前限制和真实链接，并保持“本人编写的程序”语义。

## 6. 海洋到星空

| 全局进度 | 事件 |
|---:|---|
| 0.66–0.69 | 水流增强 |
| 0.69–0.72 | 镜头上升 |
| 0.72–0.75 | 三层浪花 |
| 0.75–0.78 | 背景翻转 |
| 0.78–0.80 | 泡沫转星点 |

同一粒子对象建议具有 underwaterState、foamState 和 starState，通过插值转换，不销毁后重建。

## 7. DOM 稳定

使用独立容器：

```text
sceneWorld
contentOverlay
fixedUI
```

仅 `sceneWorld` 翻转。导航、进度、正文和关键按钮不参与背景旋转。

## 8. 输入

支持鼠标滚轮、触控、触控板、键盘 PageDown/PageUp 和进度节点点击。

禁止禁用默认滚动、长时间阻塞主线程、强制整页吸附和无限惯性滞后。

## 9. Reduced Motion

```text
陆地静态场景
→ 像素遮罩
→ 深海静态场景
→ 像素遮罩
→ 星空静态场景
```

取消大幅旋转、快速视差、镜头摇晃和大规模粒子涌动。
