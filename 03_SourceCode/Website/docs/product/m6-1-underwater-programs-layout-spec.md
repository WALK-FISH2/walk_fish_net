# M6.1 深海 Programs 左右分栏与标题渐隐规格

状态：已完成实现与正式验收  
日期：2026-07-23

## 1. 背景

当前首页深海 Programs 区域中的“沉在海里的程序档案”标题、三张档案卡片和“打开全部程序”入口均使用普通文档流。该实现解决了旧版 sticky 堆栈造成的标题/卡片及卡片/卡片重叠，但桌面端标题会与卡片一起离开视口，没有形成左侧稳定的阅读锚点。

M6.1 在不恢复旧堆栈、不修改 Program 内容模型和路由的前提下，重新组织桌面端 Programs 内容层：标题位于左上阅读锚点并在 section 内停驻，三张卡片整体移到右侧，但继续保持当前完整、连续、互不重叠的纵向排列。最后一张卡片划过时，标题提前变暗并逐渐消失，视觉上融入即将到来的夜空。移动端继续保留当前稳定的单列顺序。

## 2. 范围

本阶段只调整首页深海 Programs 内容层的左右位置、标题停驻与标题退场效果，以及相关验收。

不得修改：

- Program 字段、内容集合、排序和文案语义；
- `/programs`、`/programs/[slug]` 及 `/projects` 兼容路由；
- M4 深海 Canvas 美术、M4.5 浪花、M5 海空过渡和 M5.5 星空视觉；
- `0.430–0.640` 的 Programs/深海阅读窗口和 `0.640` 的 M5 边界；
- Canvas 世界固定 `0°` 的约束；
- 首页其他章节、导航和文章系统。

## 3. 当前实现事实

- Programs DOM 位于 `src/components/ImmersiveHome.tsx` 的 `.story-stage--programs`；
- 标题位于 `.stage-copy`，三张卡片位于 `.portholes`，单卡为 `.porthole`；
- 当前标题与卡片均为 `position: relative`，没有发生停驻；
- 当前移动端已经是“标题在上、三张卡片在下”的单列结构，M6.1 应保留这一基础表现；
- 旧版按 `--porthole-index` 递增 sticky top 的堆栈已经因重叠问题移除，不得恢复。

## 4. 桌面端目标布局

桌面完整动画使用两条互不侵占的阅读轨道：

```text
┌────────────────────────────────────────────────────┐
│ 左侧标题轨道          安全间距          右侧卡片轨道 │
│                                                    │
│ 沉在海里的程序档案                    档案卡片 01   │
│                                      档案卡片 02   │
│                                      档案卡片 03   │
│ 标题渐暗并隐入夜空                   打开全部程序   │
└────────────────────────────────────────────────────┘
```

### 4.1 标题

- 标题块位于 Programs section 的左上区域，不居中，也不跟随卡片移到右侧；
- 标题在进入 Programs 主要阅读窗口后停驻，在最后一张卡片通过阅读区时退出；
- 停驻属于 section 内受限的 sticky，不得成为跨章节的 viewport fixed 元素；
- 标题顶部必须避开浮动导航，建议初始配置为 `--program-title-safe-top: clamp(112px, 16vh, 176px)`；
- 标题轨道建议宽度为 `300–380px`，正文允许自然换行；
- 标题与右侧任一卡片在全部正向、反向滚动位置都不得发生矩形相交。

### 4.2 三张卡片

- 三张卡片整体向右移动，进入右侧卡片轨道；
- DOM 和键盘顺序保持卡片 01 → 02 → 03；
- `.portholes` 继续使用当前完整的纵向 Grid/普通文档流，三张 `.porthole` 同时存在并按现有顺序完整排列；
- 不为单张卡片增加 sticky、fixed、独立步骤容器、接棒动画或“前卡消失后后卡进入”的显示逻辑；
- 卡片继续随页面自然向上划过，出场方式、相邻间距和三种卡片之间的相对节奏尽量保持当前效果；
- 卡片之间继续依靠现有显式 gap 隔离，不通过叠放、降低透明度、裁掉正文或隐藏链接制造切换效果；
- 建议右侧轨道宽度为 `640–820px`，左右轨道间距为 `clamp(48px, 6vw, 96px)`；
- 右移量只需达到标题与任一卡片不重叠，同时不得导致视口横向溢出；实际值在实现时根据 1280×720 与 1920×1080 集中配置；
- terminal、probe、capsule 三种外形、内容字段和真实链接保持不变。

### 4.3 标题退场与“隐入夜空”

- 标题不能等到最后一帧突然消失；
- 当最后一张卡片进入 Programs 尾段时，标题开始从深海亮色逐步变为夜蓝/暗靛色，并同步降低透明度；
- 当最后一张卡片划过并离开主要阅读区时，标题透明度到达 `0`，释放 sticky，不继续覆盖 M5 或星空内容；
- 建议将标题退场集中为独立变量，例如 `--program-title-exit-progress` 和 `--program-title-exit-opacity`，不得复用单卡索引制造分段效果；
- 渐暗过程允许非常轻的 `0–12px` 向上漂移，但禁止旋转、大范围 blur、缩放跳变或闪烁；
- 向上滚动时必须严格倒放：标题从夜空色逐渐恢复为深海亮色，并回到左上阅读锚点；
- 三张卡片不参与该标题专属渐暗；卡片继续沿用当前 Programs 整体退场样式。

### 4.4 桌面端适用条件

- 两栏布局只在内容宽度足以同时容纳标题轨道、间距和卡片轨道时启用；
- 当视口不足以保证两栏安全距离时，必须切换为单列顺序流，不得压缩到标题和卡片相交；
- 实现不得新增第二个滚动状态机、整页 scroll lock、强制吸附或额外的页面级 ScrollTrigger；
- 只有标题使用 section 内 CSS sticky；卡片继续使用普通文档流。标题渐暗仍读取现有 `globalProgress` 或由它派生的单一局部进度。

## 5. 移动端目标布局

移动端竖屏继续沿用当前基本满足需求的单列结构：

```text
沉在海里的程序档案
说明

档案卡片 01
档案卡片 02
档案卡片 03
打开全部程序
```

- 375px 下标题在上，三张卡片在下；
- 标题和卡片均使用普通文档流，不启用桌面停驻；
- 三张卡片依次排列，间距明确，互不重叠；
- 卡片宽度不超过内容容器，不产生页面级横向滚动；
- 不改变当前卡片内容、链接顺序和触控目标；
- 中间宽度若无法安全容纳桌面双轨，也采用同一单列降级原则。

## 6. 简化动画与降级

- 简化动画保留当前稳定的普通文档流：标题在上，三张卡片在下；
- 简化动画不执行桌面标题停驻或标题专属渐暗，继续使用既有 Programs 整体退出；
- Canvas fallback 下仍显示完整标题、三张卡片和“打开全部程序”入口；
- 完整/简化模式切换不得把用户卡在 Programs section，也不得改变卡片顺序；
- 向上滚动时，三张卡片继续随普通文档流自然返回，标题的渐暗与消失过程反向恢复。

## 7. 无障碍与内容边界

- DOM 源码顺序保持标题 → 卡片 01 → 卡片 02 → 卡片 03 → 总入口；
- CSS 视觉分栏不得改变屏幕阅读器和 Tab 顺序；
- 所有卡片标题与操作链接始终可见、可点击、可聚焦；
- sticky 元素不得覆盖当前卡片链接、动效模式开关、阶段导航或浮动导航；
- Program 继续表示站点所有者本人编写的程序，不改回 Project/projects 语义。

## 8. 正式验收

桌面完整动画至少验证：

```text
43%  Programs 区域进入前，标题和卡片尚未错误占位
46%  标题进入左上安全锚点，完整卡片流位于右侧
51%  卡片保持当前纵向排列和自然滚动，标题位置稳定
56%  三张卡片仍为同一完整文档流，没有逐张隐藏或接棒
59%  最后一张卡片进入尾段，标题开始变暗
61%  标题透明度继续降低，最后一张卡片和总入口仍可读
64%  标题随最后一张卡片退出而完全隐去，M5 边界不变
61% → 43% 完整反向滚动
```

还必须验证：

- 1280×720 与 1920×1080；
- 375×812：标题在上、三张卡片在下；
- 完整动画与简化动画；
- 运行中切换完整/简化后仍可继续到 100% 并回到 0%；
- 键盘 Tab 顺序、指针点击和真实 Program 路由；
- Canvas fallback；
- 浏览器控制台；
- 标题/卡片、卡片/卡片以及标题/fixed UI 的矩形相交均为 0；当前主阅读卡片不得被 fixed UI 遮挡。卡片采用普通文档流自然离场时，已经离开主要阅读区的边缘片段允许从浮动导航下方经过，但不得覆盖导航、拦截导航交互或形成持续遮挡。
- 标题透明度在退场阶段单调下降，反向滚动时单调恢复。

## 9. 完成条件

只有以下条件全部有代码、测试或浏览器证据时，M6.1 才能标为完成：

- 桌面标题稳定停驻在左上安全区；
- 三张卡片位于右侧并保持当前完整纵向排列与自然出场；
- 标题与卡片、卡片与卡片没有重叠；
- 最后一张卡片划过时标题先变暗、再完全隐入夜空，反向滚动可恢复；
- 移动端保持标题在上、三张卡片在下；
- 简化动画和 Canvas fallback 内容完整；
- 正向、反向和模式切换均不中断滚动；
- M4、M4.5、M5、M5.5、M6 基线无回归；
- Astro Check、ESLint、测试、生产构建和静态路由验证通过。
