# Changelog

## [Unreleased]

### Added

- 建立完整 Vibe Coding 文档体系；
- 明确“做点啥呢”为本人编写的程序示例；
- 增加 Programs 领域模型；
- 增加程序演示隔离决策；
- 增加静态导出审查阶段；
- 增加三世界美术和海洋到星空过渡任务。
- 增加 M5.5 独立高速流星、行星多层光环和透明浪花/气泡生产素材。
- 增加 M6 固定完整/简化动画开关、系统简化建议和统一动效偏好状态。

### Changed

- 将泛化的“项目”语义逐步收敛为“程序”；
- 将第一轮骨架标记为“已汇报完成，待独立验证”。
- 将陆地世界的远景、中景、近景、前景视差强度及层内倍率集中到故事配置，并完成 M3 正式验收。
- 将下潜时间线、深海四层视差和 `0.640–0.660` 预热集中到故事配置，补全下潜镜头、水面退远、固定鱼群/气泡池与 Programs 档案信息，并完成 M4 正式验收。
- 将陆海交接精修为独立远浪、中浪、前浪、固定泡沫池和镜头穿浪序列，补全移动端简化与 Reduced Motion 静态浪幕，并完成 M4.5 正式验收。
- 按需求更正将“背景世界翻转”从 M5 移除并计入 M4.5：所有模式下 Canvas 世界固定为 0°，海洋到星空仅通过颜色、光束、透明度和粒子形态平滑过渡。
- 在不改变 M5 时间线和章节高度的前提下，将固定流星修正为朝右下；三层陆海浪花改用处理后的透明 sprite 主轮廓，并保留原 Graphics 降级边界。
- 经项目所有者实际试用，将 M5.5 独立动态流星的随机生成间隔由初始建议的 5–14 秒调整为 1–3 秒，并同步回归测试与工程文档。
- 将首页产品默认动效改为完整动画；保留 `?motion=full|reduce` 临时覆盖，并以“URL > 已保存选择 > 完整默认值”解析最终模式。

### Fixed

- 修复响应式视口切换或重载时，`SceneController.resize()` 可能早于 Pixi renderer 初始化而产生控制台错误的问题。
- 修复 `?motion=full` 验收参数切换故事高度后未刷新 ScrollTrigger，导致移动端验收进度与画面进度不一致的问题。
- 修复陆地 Hero 与三张文章路牌在旅行者行走路径上持续遮挡人物的问题：为 Hero 设置响应式顶部安全偏移，并集中调整路牌顶部、卡片高度、错层和路牌柱参数；短视口使用紧凑卡片，`≤980px` 使用横向路牌轨道。
- 修复 Programs 标题与档案卡片、相邻档案卡片在标准动效和 `?motion=full` 下因 sticky 错位而持续重叠的问题；同时补齐强制完整动效模式的桌面与移动端阶段高度。
- 修复完整/简化模式切换时因阶段高度变化和 GSAP refresh 造成的滚动跳位；切换后在刷新完成的下一帧恢复等价故事进度。

### Security

- 规定程序演示沙箱与前端秘密处理边界。
